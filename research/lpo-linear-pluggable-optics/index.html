Linear Pluggable Optics / Linear Drive Pluggable Optics

一、核心判断

LPO 是对传统可插拔光模块的一次“减法”：尽量去掉模块内 DSP/retimer，让 host ASIC 的 SerDes、线性驱动/跨阻放大器和光引擎共同完成链路。它不是 CPO，也不是相干光；它瞄准的是数据中心内部短距 800G/1.6T IM-DD 光互连，目标是在保持可插拔运维形态的同时，降低功耗、时延和模块成本。

传统 800G 光模块通常在模块内放置 DSP，用来做 CDR、均衡、FEC 协同、信号整形和链路容错。DSP 提高了互通性和链路鲁棒性，但也带来明显功耗、成本和时延。LPO 的思路是：如果链路距离较短、host ASIC SerDes 足够强、系统厂商能控制交换机板级通道和模块参数，就可以把部分数字信号处理从模块里拿掉。

因此 LPO 更像 hyperscaler 和高端交换系统的“系统级优化”，而不是一个单独靠模块厂就能完成的产品。它要求交换 ASIC、PCB 通道、连接器、光模块、线缆、测试仪表和链路训练算法一起设计。

四、为什么 AI 数据中心关心 LPO

  • 功耗：800G/1.6T 端口数量快速增加，模块内 DSP 的瓦数会被交换机端口数放大成整机散热问题。
  • 时延：DSP 会引入额外处理时延。训练网络的尾延迟和同步通信对时延敏感，短距链路更愿意用简单路径。
  • 成本：去 DSP 可以降低模块 BOM，但节省的一部分成本会转移到 host SerDes、板级设计和测试复杂度上。
  • 可插拔：LPO 仍保留 QSFP-DD/OSFP 这类运维习惯，比 CPO 更容易被数据中心运维体系接受。

六、成熟度与观察指标

成熟度判断 LPO 比 CPO 更接近当前可插拔生态，但量产节奏取决于 800G/1.6T 系统互通和 hyperscaler 是否愿意接受更紧的链路预算。它可能先在短距、同厂商闭环、云厂商定制环境里落地。

  • 观察 800G/1.6T LPO 是否进入云厂商规模采购，而不仅是 OFC/ECOC 展示。
  • 观察 LPO、LRO、传统 DSP 模块之间的功耗差、良率和现场故障率。
  • 观察交换 ASIC 是否明确为 LPO 优化 SerDes 和链路训练。
  • 观察 200G/lane 时代 LPO 的链路余量是否仍然足够。

七、与现有 AIDC 主线的关联

LPO 是 AIDC 光互连里最现实的“降功耗中间路线”之一。它不改变前面板模块形态，因此比 CPO 更容易在现有交换机和数据中心运维流程里推进；但它也不具备 CPO 那种从根本上缩短 ASIC 到光引擎电距离的长期潜力。可以把 LPO 看作 800G/1.6T 可插拔阶段的效率优化，把 CPO 看作交换系统架构的下一次迁移。
