# Goal

从 akshare 采集个股日频筹码分布数据，存入 DuckDB，供分析引擎（REQ-037）和
仪表盘（REQ-039）消费。建立筹码数据的标准化 schema 和采集接口。

# In Scope

## DuckDB Schema 扩展 — `engine/data_agent/storage.py`

在现有 `init_db()` 中新增两张表：

```sql
CREATE TABLE IF NOT EXISTS chip_distribution (
    code        TEXT    NOT NULL,
    date        DATE    NOT NULL,
    price_lower REAL    NOT NULL,  -- 该档位价格下界
    price_upper REAL    NOT NULL,  -- 该档位价格上界
    chip_ratio  REAL    NOT NULL,  -- 该档位筹码占比（0–1）
    PRIMARY KEY (code, date, price_lower)
);

CREATE TABLE IF NOT EXISTS chip_summary (
    code           TEXT  NOT NULL,
    date           DATE  NOT NULL,
    avg_cost       REAL  NOT NULL,  -- 市场平均持仓成本
    profitable_pct REAL  NOT NULL,  -- 收盘获利筹码比例（0–1）
    concentration  REAL  NOT NULL,  -- 集中度（0–100）
    range_70_lower REAL  NOT NULL,  -- 70% 筹码区间下界
    range_70_upper REAL  NOT NULL,  -- 70% 筹码区间上界
    range_90_lower REAL  NOT NULL,  -- 90% 筹码区间下界
    range_90_upper REAL  NOT NULL,  -- 90% 筹码区间上界
    PRIMARY KEY (code, date)
);
```

## Out of Scope

- 批量采集多股票（一期单股按需采集）
- 历史筹码数据回填（仅当日/最新）
- 筹码变化趋势对比（REQ-037 信号层处理）

# Acceptance Criteria

- `TC-036-01`：`fetch_chip_summary('688143')` 返回 `ChipSummary`，`avg_cost > 0`，`0 < profitable_pct ≤ 1`，`len(bars) > 0`
- `TC-036-02`：`store_chip_data(conn, summary)` 后，`SELECT * FROM chip_summary WHERE code='688143'` 返回至少 1 行，各字段类型正确
- `TC-036-03`：`store_chip_data` 对同一 (code, date) 二次写入不报错（UPSERT 语义）
- `TC-036-04`：akshare 返回空 DataFrame 时，`fetch_chip_summary` 抛出 `ChipDataUnavailable`，不返回 None

# Implementation Notes

- akshare 调用：`ak.stock_cyq_em(symbol=code)`，注意科创板代码需确认是否需要 `688143` vs `sh688143` 格式
- `concentration` 字段在 akshare 返回列中可能名为 `集中度` 或需自行从 70%/90% 区间宽度推算；实现时以实际返回列为准
- `bars` 数据量可能较大（数十到数百档位），写入 `chip_distribution` 时批量 INSERT
- 遵循 `engine/agent/technical_fetcher.py` 的错误处理和日志风格
