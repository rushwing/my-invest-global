# Goal

从 akshare 采集个股日频筹码分布数据，存入 DuckDB，供分析引擎（REQ-037）和
仪表盘（REQ-039）消费。建立筹码数据的标准化 schema 和采集接口。

# In Scope

## DuckDB Schema 扩展 — `engine/data_agent/storage.py`

在现有 `init_db()` 中新增两张表：

```sql
CREATE TABLE IF NOT EXISTS chip_distribution (
    code        TEXT    NOT NULL,
    date        DATE    NOT NULL,
    price_lower REAL    NOT NULL,  -- 该档位价格下界
    price_upper REAL    NOT NULL,  -- 该档位价格上界
    chip_ratio  REAL    NOT NULL,  -- 该档位筹码占比（0–1）
    PRIMARY KEY (code, date, price_lower)
);

CREATE TABLE IF NOT EXISTS chip_summary (
    code           TEXT  NOT NULL,
    date           DATE  NOT NULL,
    avg_cost       REAL  NOT NULL,  -- 市场平均持仓成本
    profitable_pct REAL  NOT NULL,  -- 收盘获利筹码比例（0–1）
    concentration  REAL  NOT NULL,  -- 集中度（0–100）
    range_70_lower REAL  NOT NULL,  -- 70% 筹码区间下界
    range_70_upper REAL  NOT NULL,  -- 70% 筹码区间上界
    range_90_lower REAL  NOT NULL,  -- 90% 筹码区间下界
    range_90_upper REAL  NOT NULL,  -- 90% 筹码区间上界
    PRIMARY KEY (code, date)
);
```

### akshare 列名映射

`ak.stock_cyq_em(symbol=code)` 返回 DataFrame，预期列名（中文）→ ChipSummary 字段映射：

| akshare 列名 | ChipSummary 字段 | 缺失时行为 |
|---|---|---|
| `获利比例` | `profitable_pct`（除以 100 归一化） | 抛 `ChipDataUnavailable` |
| `平均成本` | `avg_cost` | 抛 `ChipDataUnavailable` |
| `90成本-低` | `range_90_lower` | 抛 `ChipDataUnavailable` |
| `90成本-高` | `range_90_upper` | 抛 `ChipDataUnavailable` |
| `70成本-低` | `range_70_lower` | fallback：使用 `range_90_lower` 值，不抛异常 |
| `70成本-高` | `range_70_upper` | fallback：使用 `range_90_upper` 值，不抛异常 |
| `集中度` | `concentration` | fallback：`(range_90_upper - range_90_lower) / avg_cost * 100`，取值截断至 `[0, 100]` |
| 逐档价格/比例 | `bars[*]` | 空列表（不抛异常，直方图不渲染）|

> 实现时以实际运行中 akshare 返回的列名为准；若列名与上表不符，
> 实现者需更新此映射表并在 PR 中注明。

## Out of Scope

- 批量采集多股票（一期单股按需采集）
- 历史筹码数据回填（仅当日/最新）
- 筹码变化趋势对比（REQ-037 信号层处理）

# Acceptance Criteria

- `TC-036-01`：mock `ak.stock_cyq_em` 返回含全部列的标准 DataFrame（基于 688143 预期值），`fetch_chip_summary('688143')` 返回 `ChipSummary`，`avg_cost > 0`，`0 < profitable_pct ≤ 1`，`len(bars) > 0`
