# Goal

实现 `UpdateClass` 八级枚举、`IndicatorGroup` 枚举（附 SourcePolicy）和
时区感知的 `MacroScheduler`，以及 BLS/NBS 发布日历预填充模块
`release_calendar.py`（BP-7）。

# In Scope

## `engine/macro_agent/update_class.py`

```python
class UpdateClass(str, Enum):
    INTRADAY_FAST = "intraday_fast"   # 1–5 min
    INTRADAY_SLOW = "intraday_slow"   # 15–60 min
    DAILY         = "daily"           # post-close
    WEEKLY        = "weekly"          # reserved
    MONTHLY_FIXED = "monthly_fixed"   # BLS/NBS 发布日
    QUARTERLY     = "quarterly"       # SEC 10-Q/10-K 受理后
    EVENT_DRIVEN  = "event_driven"    # FOMC 决议日等
    STATIC_YAML   = "static_yaml"     # 人工维护
```

## `engine/macro_agent/indicator_groups.py`

- `IndicatorGroup`（str Enum）：`MACRO_ECON`、`COMMODITIES`、`US_INDICES`、
  `CLOUD_CAPEX`、`APAC_SEMI`、`CHINA_MARKET`、`GEOPOLITICAL`
  （对应 Groups I/J/K/L/M/N/O）
- `IndicatorConfig` dataclass：
  ```python
  @dataclass
  class IndicatorConfig:
      indicator_id: str
      group: IndicatorGroup
      update_class: UpdateClass
      primary_source: str
      backups: list[str]
      market_tz: str            # "America/New_York" / "Asia/Shanghai" 等
      session_hours: tuple[time, time] | None  # None = 全天（FX/期货）
      poll_interval_s: int      # INTRADAY_FAST=60-300, INTRADAY_SLOW=900-3600
      timeout_s: int = 20
      cooldown_s: int = 600
  ```
- `INDICATOR_CONFIGS: dict[str, IndicatorConfig]`：30+ 指标完整配置
  （对应 ADR-007 决策四的完整分类表）

## `engine/macro_agent/scheduler.py`

```python
class MacroScheduler:
    def __init__(self, storage: MacroStorage, release_calendar: ReleaseCalendar) -> None: ...

    def get_due_indicators(
        self,
        now: datetime | None = None,
        groups_filter: list[str] | None = None,
    ) -> list[IndicatorConfig]:
        """
        返回当前时刻应触发采集的指标列表。
        groups_filter 非 None 时只返回 IndicatorConfig.group.value 在列表中的指标。
        判断逻辑按 UpdateClass：
          INTRADAY_FAST/SLOW：检查对应市场时区的交易时段窗口，
            且距上次成功采集超过 poll_interval_s
          DAILY：检查所有相关市场当日是否已收盘且尚未采集
          MONTHLY_FIXED：检查今日是否在 release_calendar 的发布日列表中；
            若 release_dates 无匹配行（日历未填充），降级为每日采集一次
          QUARTERLY：由外部 EDGAR 接受事件触发，不在此处轮询
          EVENT_DRIVEN：由外部事件触发（FOMC 决议日来自 fomc_calendar 表）
          STATIC_YAML：不触发（人工维护）
        """
```

- 时区判断使用 `zoneinfo.ZoneInfo`（Python 3.9+ 标准库）
- 交易时段配置在 `IndicatorConfig.session_hours` 中逐指标定义，不使用全局
  A 股日历
- 查询 `storage.get_latest_indicator()` 确定上次采集时间

## `engine/macro_agent/release_calendar.py`

```python
class ReleaseCalendar:
    def __init__(self, storage: MacroStorage) -> None: ...

    def populate_bls(self, year: int) -> None:
        """从 BLS 网站解析当年 CPI/PPI 发布日期，写入 DuckDB release_dates 表。
        解析失败时 log warning，保留已有数据（安全降级）。"""

    def populate_nbs(self, year: int) -> None:
        """从 NBS 网站解析当年中国 CPI/PPI 发布日期。"""

    def is_release_day(self, indicator_id: str, date: date) -> bool:
        """判断 date 是否是 indicator_id 的官方发布日。"""
```

新增辅助 DuckDB 表：
```sql
CREATE TABLE IF NOT EXISTS release_dates (
    indicator_id    TEXT    NOT NULL,
    release_date    DATE    NOT NULL,
    source          TEXT,   -- "BLS" / "NBS"
    PRIMARY KEY (indicator_id, release_date)
);
```

# Acceptance Criteria

- `TC-008-01 INTRADAY_FAST 时段外不触发`：模拟时间为 01:00 ET（NYSE 休市），
  `get_due_indicators()` 不返回 `UpdateClass.INTRADAY_FAST` 的 NVDA 配置
- `TC-008-02 INTRADAY_FAST 时段内触发`：模拟时间为 10:00 ET，NVDA 上次
  采集超 300s，返回 NVDA 配置
- `TC-008-03 MONTHLY_FIXED 发布日触发`：`release_calendar.is_release_day(
  "CPIAUCSL", bls_release_date)` 返回 True；非发布日返回 False
- `TC-008-04 DAILY 未采集触发`：模拟 KRX 收盘后（16:00 KST），Samsung
  当日未采集，返回 Samsung 配置
- `TC-008-05 populate_bls 解析失败降级`：mock HTTP 500，`populate_bls()`
  不 raise，log warning，已有 release_dates 数据不丢失
- `TC-008-06 INDICATOR_CONFIGS 完整性`：所有 ADR-007 分类表中的指标均有对应
  `IndicatorConfig`，且 `update_class` 字段与表中一致
- `TC-008-07 MONTHLY_FIXED 日历缺失降级`：`release_dates` 表无 CPIAUCSL 记录时，
  模拟时间为当日首次调用，`get_due_indicators()` 仍返回 CPIAUCSL 配置（每日降级触发）；
  当日已触发过一次时，不重复返回（防止无日历时频繁触发）

# Implementation Notes

- `get_due_indicators()` 以 `now` 参数支持时间注入（测试可控性）
- `IndicatorConfig.session_hours` 的时区与 `market_tz` 对应；
  例如 NYSE 为 `(time(9, 30), time(16, 0))` 配合 `"America/New_York"`
- BLS 发布日历 URL 及解析逻辑若在实现时官网结构已变，改为解析 BLS
  公开 API（`https://www.bls.gov/schedule/2026/home.htm`）
