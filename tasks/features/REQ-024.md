# Goal

基于 `langchain-neo4j` 的 `GraphCypherQAChain`，封装只读 KG 查询接口，供 Phase 9
LangGraph ReAct Agent 调用。所有查询通过 QAChain 生成 Cypher，不暴露原始 Cypher API
（防 prompt injection）。返回结构化 `KGQueryResult`，包含生成的 Cypher、执行结果和
自然语言答案。

# In Scope

## `engine/kg/query.py`

round 2 反馈核心问题：`GraphCypherQAChain.invoke()` 在返回前已经执行了生成的 Cypher，
后处理拦截无法阻止写操作实际发生。

**新设计：分离 Cypher 生成与执行（两步调用）**

不使用 `GraphCypherQAChain` 的一体化 `invoke()`，改为：
1. 用 LLM 生成 Cypher（只 prompt，不执行）
2. 校验 Cypher 不含写关键字——**如果含写关键字，立即返回 error，不进行任何 DB 调用**
3. 只对通过校验的 Cypher 调用 `Session.run()`

```python
from __future__ import annotations
import re
from pydantic import BaseModel
from langchain_core.prompts import ChatPromptTemplate
from langchain_anthropic import ChatAnthropic
from langchain_neo4j import Neo4jGraph
from engine.kg.connection import KGConnection

_WRITE_PATTERN = re.compile(
    r'\b(CREATE|MERGE|DELETE|SET|REMOVE|DROP|DETACH)\b',
    re.IGNORECASE,
)

_CYPHER_GENERATION_PROMPT = ChatPromptTemplate.from_template(
    "You are a Neo4j Cypher expert. Based on the schema below, write a Cypher MATCH query "
    "that answers the question. Return ONLY the Cypher statement, nothing else.\n\n"
    "Schema:\n{schema}\n\n"
    "Question: {question}\n\n"
    "Cypher:"
)

_ANSWER_PROMPT = ChatPromptTemplate.from_template(
    "Given the following Cypher query results, answer the question in Chinese.\n\n"
    "Question: {question}\n"
    "Results: {results}\n\n"
    "Answer:"
)

class KGQueryResult(BaseModel):
    question: str
    cypher_used: str
    raw_results: list[dict]
    answer: str
    error: str | None = None

def _is_write_cypher(cypher: str) -> bool:
    return bool(_WRITE_PATTERN.search(cypher))

class KGQuerier:
    def __init__(
        self,
        conn: KGConnection,
        model: str = "claude-haiku-4-5-20251001",
    ) -> None:
        self._conn = conn
        self._model = model
        self._schema: str | None = None
        self._llm: ChatAnthropic | None = None

    def _get_llm(self) -> ChatAnthropic:
        if self._llm is None:
            self._llm = ChatAnthropic(model=self._model, max_tokens=512)
        return self._llm

    def _get_schema(self) -> str:
        if self._schema is None:
            graph = Neo4jGraph(
                url=self._conn._settings.neo4j_uri,
                username=self._conn._settings.neo4j_user,
                password=self._conn._settings.neo4j_password,
            )
            self._schema = graph.schema
        return self._schema

    def _generate_cypher(self, question: str) -> str:
        chain = _CYPHER_GENERATION_PROMPT | self._get_llm()
        result = chain.invoke({"schema": self._get_schema(), "question": question})
        return result.content.strip()

    def _run_cypher(self, cypher: str) -> list[dict]:
        with self._conn.driver().session() as session:
            result = session.run(cypher)
            return [dict(r) for r in result]

    def _summarize(self, question: str, results: list[dict]) -> str:
        chain = _ANSWER_PROMPT | self._get_llm()
        resp = chain.invoke({"question": question, "results": str(results)})
        return resp.content.strip()

    def query(
        self,
        question: str,
        *,
        llm: ChatAnthropic | None = None,
        schema: str | None = None,
        session_runner=None,
    ) -> KGQueryResult:
        """
        Injecting `llm`, `schema`, `session_runner` allows full unit-test coverage
        without a live Neo4j connection.

        Pre-execution read-only guard: _is_write_cypher() is checked BEFORE any
        DB call. If it fires, session_runner / _run_cypher() is never called.
        """
        try:
            if llm is not None:
                self._llm = llm
            if schema is not None:
                self._schema = schema

            cypher = self._generate_cypher(question)

            if _is_write_cypher(cypher):
                return KGQueryResult(
                    question=question,
                    cypher_used=cypher,
                    raw_results=[],
                    answer="",
                    error="read_only_violation",
                )

            run = session_runner if session_runner is not None else self._run_cypher
            raw = run(cypher)
            answer = self._summarize(question, raw)
            return KGQueryResult(
                question=question,
                cypher_used=cypher,
                raw_results=raw if isinstance(raw, list) else [],
                answer=answer,
            )
        except Exception as exc:
            return KGQueryResult(
                question=question,
                cypher_used="",
                raw_results=[],
                answer="",
                error=str(exc),
            )
```

**只读保证**：`session_runner`（即 `_run_cypher`）在 `_is_write_cypher` 通过后才被调用，
不存在"已执行写语句再检查"的问题。测试用 `session_runner=mock_fn` 注入，无需真实 Neo4j。

- `TC-024-02`：mock llm 在生成 Cypher 时抛 `ValueError("timeout")` →
  `result.error == "timeout"`，`result.answer == ""`，无异常传播
- `TC-024-03`：mock llm 生成写 Cypher（含 `MERGE`）→ `result.error == "read_only_violation"`，
  `session_runner` **从未被调用**（DB 调用在校验前被阻断）
- `TC-024-04`：`_is_write_cypher("MATCH (n) RETURN n")` 返回 `False`；
  `_is_write_cypher("MATCH (n) DELETE n")` 返回 `True`；
  `_is_write_cypher("MATCH (n) SET n.x=1")` 返回 `True`
- `TC-024-05`：注入 `llm`, `schema`, `session_runner` 后 `query()` 不触发真实 `_get_schema()`
  也不触发真实 `_run_cypher()`（完全不需要 Neo4j 连接）

# Implementation Notes

- **为何放弃 `GraphCypherQAChain.invoke()`**：该 chain 在一次调用内完成"生成 Cypher → 执行 →
  生成摘要"，无法在执行前插入校验点；改为手动三步（生成 / 校验 / 执行）后，校验发生在任何 DB 调用之前
- `_WRITE_PATTERN` 包含 `SET`：`ON CREATE SET` / `ON MATCH SET` 属于 `MERGE` 的子句，
  在纯读场景不会出现；包含 `SET` 保持规则简单，宁可误判少量边缘 Cypher 也不漏过写语句
- `langchain-neo4j` 仍用于 schema 读取（`Neo4jGraph.schema`），不再用于 Cypher 执行
- `pyproject.toml` 依赖：去掉 `langchain-neo4j` 的 `allow_dangerous_requests` 依赖路径，
  但仍需 `langchain-neo4j>=0.3` 读取 schema
- 使用 `claude-haiku-4-5-20251001` 而非 Sonnet 生成 Cypher，降低 KG 查询的 token 成本
- Phase 9 集成：LangGraph `kg_retrieval` 节点调用 `KGQuerier.query()`，结果存入
  `AnalysisState.kg_subgraph`

# Req Review Feedback (round 1)

- 需求标题和 Goal 承诺“只读查询接口”，但当前设计没有实际只读防线：`GraphCypherQAChain`
  使用 `allow_dangerous_requests=True`，并沿用 REQ-021 的 `neo4j` 账号；该账号默认具备写权限。
  prompt 生成的 `CREATE` / `MERGE` / `DELETE` Cypher 可能被执行，和 Out of Scope 的“写操作”
  冲突。
- 请在需求中补齐一种可验证机制：例如新增只读 Neo4j 用户/环境变量并要求 `_build_chain()` 使用
  只读凭据，或在执行前校验生成 Cypher 只允许 `MATCH` / `RETURN` 等读语句。对应 Acceptance
  Criteria 也需要覆盖“危险写 Cypher 被拒绝或不会执行”。

# Req Review Feedback (round 2)

- 本轮新增的 `_is_write_cypher` 是在 `chain.invoke()` 返回后检查 `intermediate_steps`，
  但 `GraphCypherQAChain.invoke()` 已经执行了生成的 Cypher。也就是说，mock 场景里可以返回
  `error='read_only_violation'`，真实 Neo4j 场景里写语句仍可能已经生效，这没有满足
  “只读查询接口”或 `TC-024-04` 中“不执行写语句”的要求。
- 请改为执行前防线，或将需求改成可实际保证的机制。可选路径：
  1. 使用 Neo4j 只读账号执行 chain，并在 REQ-021/024 中补齐只读凭据配置；或
  2. 不直接使用会自动执行的 `GraphCypherQAChain.invoke()` 路径，改成先生成 Cypher、先校验、
     再只对通过校验的 Cypher 执行读取；或
  3. 明确此 REQ 不承诺“不会执行写语句”，但这样需要改标题、Goal、Out of Scope 和 AC。
