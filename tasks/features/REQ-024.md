# Goal

基于 `langchain-neo4j` 的 `GraphCypherQAChain`，封装只读 KG 查询接口，供 Phase 9
LangGraph ReAct Agent 调用。所有查询通过 QAChain 生成 Cypher，不暴露原始 Cypher API
（防 prompt injection）。返回结构化 `KGQueryResult`，包含生成的 Cypher、执行结果和
自然语言答案。

# In Scope

## `engine/kg/query.py`

```python
from __future__ import annotations
import re
from pydantic import BaseModel
from langchain_neo4j import Neo4jGraph, GraphCypherQAChain
from langchain_anthropic import ChatAnthropic
from engine.kg.connection import KGConnection

# Cypher write keywords — any generated query containing these is rejected
_WRITE_PATTERN = re.compile(
    r'\b(CREATE|MERGE|DELETE|SET|REMOVE|DROP|DETACH)\b',
    re.IGNORECASE,
)

class KGQueryResult(BaseModel):
    question: str
    cypher_used: str
    raw_results: list[dict]
    answer: str
    error: str | None = None

def _is_write_cypher(cypher: str) -> bool:
    """Return True if the Cypher string contains any write keyword."""
    return bool(_WRITE_PATTERN.search(cypher))

class KGQuerier:
    def __init__(
        self,
        conn: KGConnection,
        model: str = "claude-haiku-4-5-20251001",
    ) -> None:
        self._conn = conn
        self._model = model
        self._chain: GraphCypherQAChain | None = None

    def _build_chain(self) -> GraphCypherQAChain:
        graph = Neo4jGraph(
            url=self._conn._settings.neo4j_uri,
            username=self._conn._settings.neo4j_user,
            password=self._conn._settings.neo4j_password,
        )
        llm = ChatAnthropic(model=self._model, max_tokens=512)
        return GraphCypherQAChain.from_llm(
            llm=llm,
            graph=graph,
            allow_dangerous_requests=True,
            verbose=False,
            return_intermediate_steps=True,
        )

    def query(self, question: str, chain: GraphCypherQAChain | None = None) -> KGQueryResult:
        """
        Accept optional `chain` injection for testing — avoids real Neo4j connection.
        Generated Cypher is checked for write keywords before the result is returned;
        any write-containing Cypher produces error='read_only_violation'.
        """
        try:
            _chain = chain or self._chain or self._build_chain()
            if chain is None and self._chain is None:
                self._chain = _chain
            result = _chain.invoke({"query": question})
            steps = result.get("intermediate_steps", [])
            cypher = steps[0].get("query", "") if steps else ""
            # Read-only guard: reject Cypher containing write operations
            if _is_write_cypher(cypher):
                return KGQueryResult(
                    question=question,
                    cypher_used=cypher,
                    raw_results=[],
                    answer="",
                    error="read_only_violation",
                )
            raw = steps[1].get("context", []) if len(steps) > 1 else []
            return KGQueryResult(
                question=question,
                cypher_used=cypher,
                raw_results=raw if isinstance(raw, list) else [],
                answer=result.get("result", ""),
            )
        except Exception as exc:
            return KGQueryResult(
                question=question,
                cypher_used="",
                raw_results=[],
                answer="",
                error=str(exc),
            )
```

**只读机制说明**：`GraphCypherQAChain` 执行 Cypher 前无法截断，因此读-拦截发生在执行后、
结果返回前：若生成的 Cypher 含写关键字，丢弃执行结果并返回 `error='read_only_violation'`。
这与 Phase 9 harness 防幻觉机制二（Pydantic 工具输出校验）一致——后处理校验优于依赖
外部账号权限配置（后者需要 Neo4j 管理员操作且难以在 CI 环境验证）。

  `result.error == "timeout"`，`result.answer == ""`，无异常传播
- `TC-024-03`：`query(q, chain=mock_chain)` 不调用 `_build_chain()`（注入 chain 绕过
  真实连接）
- `TC-024-04`：mock chain 的 `intermediate_steps[0]["query"]` 为
  `"MERGE (s:Stock {code: '300308'}) RETURN s"` →
  `result.error == "read_only_violation"`，`result.raw_results == []`，`result.answer == ""`
- `TC-024-05`：`_is_write_cypher("MATCH (n) RETURN n")` 返回 `False`；
  `_is_write_cypher("MATCH (n) DELETE n")` 返回 `True`；
  `_is_write_cypher("MATCH (n) SET n.x=1")` 返回 `True`

# Implementation Notes

- `allow_dangerous_requests=True` 是 `langchain-neo4j` 的必要参数（库的安全提示，实际
  只读由 `_is_write_cypher` 后处理拦截保障，而非依赖 Neo4j 账号权限）
- 后处理拦截 vs 账号权限：选择后处理而非只读 Neo4j 账号，因为只读账号需要管理员创建 Neo4j
  用户（`CREATE USER`），在 CI 容器和开发环境中难以标准化；后处理检查是纯 Python，可在
  单元测试中完整覆盖
- `_WRITE_PATTERN` 包含 `SET` 关键字——`ON CREATE SET`/`ON MATCH SET` 只在 `MERGE` 内使用，
  而纯 `SET` 语句是写操作；两种情况都被拒绝，以保持规则简单
- 使用 `claude-haiku-4-5-20251001` 而非 Sonnet 生成 Cypher，降低 KG 查询的 token 成本
- Phase 9 集成：LangGraph `kg_retrieval` 节点调用 `KGQuerier.query()`，结果存入
  `AnalysisState.kg_subgraph`

# Req Review Feedback (round 1)

- 需求标题和 Goal 承诺“只读查询接口”，但当前设计没有实际只读防线：`GraphCypherQAChain`
  使用 `allow_dangerous_requests=True`，并沿用 REQ-021 的 `neo4j` 账号；该账号默认具备写权限。
  prompt 生成的 `CREATE` / `MERGE` / `DELETE` Cypher 可能被执行，和 Out of Scope 的“写操作”
  冲突。

  只读凭据，或在执行前校验生成 Cypher 只允许 `MATCH` / `RETURN` 等读语句。对应 Acceptance
  Criteria 也需要覆盖“危险写 Cypher 被拒绝或不会执行”。
