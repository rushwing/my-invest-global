# Goal

实现 `macro_state` 的枚举类型定义与本地持久化，使 `run_daily_pipeline()` 能读取当前宏观状态，并支持人工通过 CLI 覆盖。

新枚举必须与现有 `engine/portfolio.py` 中 `check_portfolio_balance()` 的参数类型（`Literal["green", "yellow", "red"]`）**直接兼容**，无需适配层。

# In Scope

- `engine/portfolio/macro_gate.py`（新建；同步创建 `engine/portfolio/__init__.py` 空文件）
- **`class MacroState(str, enum.Enum)`**：值为小写字符串，`GREEN = "green"` / `YELLOW = "yellow"` / `RED = "red"`
  - 继承 `str` 使 `MacroState.GREEN == "green"` 为 `True`，可直接传入 `check_portfolio_balance(macro_state=MacroState.GREEN)`，无类型错误
  - JSON 文件中存储小写字符串：`{"state": "green"}`
- `get_macro_state() -> MacroState`：读 `data/cache/macro_state.json`，键 `"state"`；文件不存在或值无效时默认返回 `MacroState.YELLOW`
- `set_macro_state_override(state: MacroState) -> None`：原子写入 JSON（先写 `.tmp` 临时文件，再 `os.replace()`），幂等
- `TARGET_ELASTIC_BY_MACRO_STATE: dict[MacroState, float]`：`{MacroState.GREEN: 0.38, MacroState.YELLOW: 0.33, MacroState.RED: 0.20}`
  - 注：`engine/portfolio.py` 中已有同名常量（string key 版本），两者共存，各自服务于不同调用方；REQ-002 在重构时统一
- CLI 入口（`if __name__ == "__main__"` 块）：`python -m engine.portfolio.macro_gate --set green|yellow|red`（小写）；无效值打印错误并 `sys.exit(1)`
- `tests/test_macro_gate.py`：见 § Test Cases

# Out of Scope

- 从新闻/CapEx数据自动解析 `macro_state`（PHASE-005 Notes 明确为后续优化项）
- `engine/portfolio/framework.py`（REQ-002 负责）
- runner 集成（REQ-004 负责）
- 统一两个 `TARGET_ELASTIC_BY_MACRO_STATE`（REQ-002 重构时处理）

# Acceptance Criteria

- `python -m engine.portfolio.macro_gate --set green` 执行后，`get_macro_state()` 返回 `MacroState.GREEN`
- `MacroState.GREEN == "green"` 为 `True`（str enum 兼容，可直接传入 `check_portfolio_balance()`）
- `python -m engine.portfolio.macro_gate --set foo` 打印错误提示并以非零退出码退出（小写无效值）
- `get_macro_state()` 在 `data/cache/macro_state.json` 不存在时不抛异常，返回 `MacroState.YELLOW`
- `get_macro_state()` 在文件内容为 `{"state": "INVALID"}` 时不抛异常，返回 `MacroState.YELLOW`（容错）
- `set_macro_state_override()` 是幂等的：对同一值调用两次不报错，结果相同
- `from engine.portfolio.macro_gate import MacroState, get_macro_state, set_macro_state_override, TARGET_ELASTIC_BY_MACRO_STATE` 全部可正常 import
- `uv run mypy engine/portfolio/macro_gate.py --strict` 通过（无类型错误）

# Implementation Notes

- `class MacroState(str, enum.Enum)` — `str` 必须在 `enum.Enum` 之前，这是 Python mixin 顺序要求
- CLI `--set` 接受小写参数（`green`/`yellow`/`red`），通过 `MacroState(value)` 构造枚举；无效值捕获 `ValueError` 后调用 `parser.error()`
- `data/cache/` 目录在 `set_macro_state_override()` 中按需创建（`Path.mkdir(parents=True, exist_ok=True)`），不依赖外部初始化
- JSON 写入：`tmp_path = cache_file.with_suffix(".tmp")` → `tmp_path.write_text(json.dumps({"state": state.value}))` → `os.replace(tmp_path, cache_file)`
- 不引入任何第三方依赖；仅使用标准库（`enum`、`json`、`pathlib`、`os`、`argparse`）
- `macro_gate.py` 是纯 Python 模块，不依赖 DuckDB / pandas / pydantic（保持依赖最小化）
- `engine/portfolio/__init__.py` 在此 REQ 中创建为空文件（仅含注释），REQ-002 负责写入实际内容
- **兼容性说明**：现有 `engine/portfolio.py` 中 `check_portfolio_balance()` 的参数类型是 `MacroState = Literal["green", "yellow", "red"]`，即字符串。`str` mixin 使本 REQ 定义的 `MacroState` enum 可直接作为该参数传入，mypy 不报错。category 字段兼容性（`"弹性股"`/"白马股"`）在 REQ-002 中处理。
