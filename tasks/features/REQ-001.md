# Goal

实现 `macro_state` 的枚举类型定义与本地持久化，使 `run_daily_pipeline()` 能读取当前宏观状态，并支持人工通过 CLI 覆盖。

**T04 修订说明（round 1）：**

evaluator-001 指出两个问题，本次修订均已解决：

1. **模块路径冲突（已解决）**：原稿将模块放在 `engine/portfolio/macro_gate.py`，而 `engine/portfolio.py` 已以文件模块形式存在，Python 无法同时解析同名的文件模块和包目录，导致 `from engine.portfolio.macro_gate import ...` 抛出 `__path__ attribute not found`。**修订：将模块路径改为 `engine/macro_gate.py`**，与 `engine/portfolio.py` 并列，互不干扰；不再创建 `engine/portfolio/__init__.py`，该操作由 REQ-002 负责。

2. **mypy Literal 兼容性（已澄清）**：`class MacroState(str, enum.Enum)` 在 **runtime** 下与 `Literal["green","yellow","red"]` 完全兼容（`MacroState.GREEN == "green"` 为 True）。但 mypy --strict 将 `MacroState.GREEN` 的静态类型识别为 `MacroState`（枚举类型），而非 `Literal["green"]`，因此直接传入 `check_portfolio_balance(macro_state=MacroState.GREEN)` 时 mypy 会报 `Argument 1 to "check_portfolio_balance" has incompatible type "MacroState"; expected "str"`。**修订：明确要求调用时使用 `.value`（`get_macro_state().value`），并在 TC 和 Implementation Notes 中注明；REQ-002 重构时将 `check_portfolio_balance` 的参数类型改为接受 `MacroState`，届时可去掉 `.value`。**

# Implementation Notes

- 模块路径：`engine/macro_gate.py`（顶层，不在子包内），`from engine.macro_gate import MacroState`
- `class MacroState(str, enum.Enum)` — `str` 必须在 `enum.Enum` 之前，这是 Python mixin 顺序要求
- **mypy 行为说明**：mypy --strict 将 `MacroState` 枚举类型视为 `str` 的子类，但不视为 `Literal["green","yellow","red"]` 的子类。直接传入 `check_portfolio_balance(macro_state=MacroState.GREEN)` 会触发 mypy 错误，因为该函数参数类型是 `Literal["green","yellow","red"]`（即 `portfolio.py` 中的类型别名）。正确做法是 `check_portfolio_balance(macro_state=get_macro_state().value)`。REQ-002 重构后，`check_portfolio_balance` 将接受 `MacroState` 类型，届时 `.value` 可去掉。
- CLI `--set` 接受小写参数（`green`/`yellow`/`red`），通过 `MacroState(value)` 构造枚举；无效值捕获 `ValueError` 后调用 `parser.error()`
- `data/cache/` 目录在 `set_macro_state_override()` 中按需创建（`Path.mkdir(parents=True, exist_ok=True)`），不依赖外部初始化
- JSON 写入：`tmp_path = cache_file.with_suffix(".tmp")` → `tmp_path.write_text(json.dumps({"state": state.value}))` → `os.replace(tmp_path, cache_file)`
- 不引入任何第三方依赖；仅使用标准库（`enum`、`json`、`pathlib`、`os`、`argparse`）
- `macro_gate.py` 不依赖 DuckDB / pandas / pydantic（保持依赖最小化）
