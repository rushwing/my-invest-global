## `engine/macro_agent/regime.py`

```python
class MacroRegime:
    def __init__(self, storage: MacroStorage) -> None: ...

    def compute(self, as_of: date | None = None) -> dict:
        """
        计算当日宏观门控状态，返回：
        {
          "as_of_date": date,
          "capex_state": "green"/"yellow"/"red"/"stale",
          "yield_curve_state": "normal"/"flat"/"inverted"/"stale",
          "risk_state": "risk_on"/"risk_off"/"neutral"/"stale",
          "composite_state": "green"/"yellow"/"red"/"stale",
          "capex_as_of": "2026-Q1",       # 时效标注
          "yield_as_of": date,
          "computed_at": datetime,
        }
        """

    def _capex_state(self, as_of: date) -> tuple[str, str]:
        """
        查询 capex_quarterly 最近 4 季 MSFT/AMZN/GOOGL/META CapEx，
        计算 Big-4 QoQ 均值（ADR-003 三色门控阈值，覆盖所有值域）：
          QoQ >= +5%          → green
          -10% <= QoQ < +5%   → yellow
          QoQ < -10%          → red
        STALE 守卫：最新季度 period_end 距 as_of 超过 180 天 → stale
        返回 (state, capex_as_of)
        """

    def _yield_curve_state(self, as_of: date) -> tuple[str, date]:
        """
        查询 macro_indicators 中 DGS10 / DGS2 最新值：
          10Y-2Y > 0.5%  → normal
          -0.2%~+0.5%    → flat
          < -0.2%        → inverted
        STALE 守卫：最新 DAILY 数据距 as_of 超过 3 个交易日 → stale
        """

    def _risk_state(self, as_of: date) -> str:
        """
        查询 macro_indicators 中 SOX (^SOX) 20 日均值趋势和
        AV_SENTIMENT 最新情绪分：
          SOX > 20d MA + 情绪 > 0.2 → risk_on
          SOX < 20d MA + 情绪 < -0.2 → risk_off
          否则 → neutral
        STALE 守卫：SOX 数据距 as_of 超过 2 个交易日 → stale
        """

    def _composite(self, capex: str, yield_curve: str, risk: str) -> str:
        """
        支柱颜色映射（stale 之外）：
          yield_curve: normal → green, flat → yellow, inverted → red
          risk:        risk_on → green, neutral → yellow, risk_off → red
          capex:       green/yellow/red（直接使用）

        合成规则（优先级从高到低）：
          1. 任一支柱为 stale → composite = stale
          2. 任一支柱映射为 red → composite = red
          3. 3/3 映射为 green → composite = green
          4. 否则 → composite = yellow
        """

    def write_cache(self, state: dict) -> None:
        """
        将 composite_state 写入 data/cache/macro_state.json（现有格式）。
        若文件已被人工编辑且不含 "auto_computed": true 字段 → 跳过写入（逃生口）。
        """
```

- `compute()` 执行完毕后调用 `MacroStorage.upsert_regime()` 写入
  `macro_regime` 表，再调用 `write_cache()`
- `MacroOrchestrator.run_once()` 在每次完整采集循环结束后调用
  `MacroRegime(storage).compute()`

## STALE 守卫设计

每个支柱维护独立的"最大允许陈旧时长"：

| 支柱 | UpdateClass | staleness_threshold |
|------|-------------|---------------------|
| CapEx | QUARTERLY | 180 天（季度约 90 天 × 2） |
| 收益率曲线 | DAILY | 3 个交易日 |
| 风险偏好 | INTRADAY_SLOW | 2 个交易日 |

任一超限 → 该支柱 state = "stale" → composite = "stale"（不传播错误判断）。

## `data/cache/macro_state.json` 格式扩展

在现有格式基础上增加 `auto_computed`、`capex_as_of`、`computed_at` 字段：

```json
{
  "state": "green",
  "auto_computed": true,
  "capex_as_of": "2026-Q1",
  "yield_as_of": "2026-05-15",
  "computed_at": "2026-05-16T08:30:00+08:00",
  "note": "自动计算；删除 auto_computed 字段以启用手动覆盖"
}
```

若文件存在且缺少 `auto_computed: true` → `write_cache()` 跳过写入，
人工覆盖生效（REQ-001 已实现的手动 CLI 逃生口保持不变）。

# Implementation Notes

- CapEx QoQ 计算：`(当季_Big4均值 - 前季_Big4均值) / 前季_Big4均值 × 100`；
  四家公司均值而非单家，避免单家财年错位影响
- SOX 20 日均值查询：`macro_indicators` 中 `indicator_id="^SOX"` 最近 20 条
  DAILY/INTRADAY_SLOW 记录的 `value` 均值（按 `period_date` 排序）
- `_composite()` 中 stale 短路优先于颜色合并，避免 stale 被误识别为 neutral
