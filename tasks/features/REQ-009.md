# Goal

实现 `MacroOrchestrator` 主循环及 Groups I（宏观经济）、J（大宗商品）、
K（美国关键指数/权益）的端到端采集，并提供 CLI 入口。

# In Scope

## `engine/macro_agent/orchestrator.py`

```python
class MacroOrchestrator:
    @classmethod
    def from_defaults(cls) -> "MacroOrchestrator":
        """工厂方法，连接所有默认 source，构建 RateLimiter、MacroStorage、MacroScheduler。"""

    def run_once(self, groups: list[str] | None = None) -> None:
        """单次采集循环：
        1. 调用 MacroScheduler.get_due_indicators(groups_filter=groups)
        2. 按 UpdateClass 分发到对应 source 的 fetch 方法
        3. 调用 MacroStorage.upsert_indicators()
        4. 写入 retrieval_log（与 stock-data-agent 共享同一表）
        5. 记录失败但不中断（下次循环重试）
        """

    def run_loop(self, poll_s: int = 60) -> None:
        """持续循环，每 poll_s 秒执行 run_once()。"""
```

**Fallback 逻辑**（与 `StockDataOrchestrator` 一致）：
- 按 `IndicatorConfig.primary_source` → `backups` 顺序逐一尝试
- 主源 `SourceError` 或 circuit open → 尝试下一个备源
- 全部失败 → log error，写 retrieval_log `status=failed`，继续下一个指标

## Groups I / J / K 采集覆盖

**Group I — 宏观经济**：
- US CPI/PPI（MONTHLY_FIXED）：`FREDSource.fetch_series("CPIAUCSL"/"PPIACO")`；
  备用 `AKShareMacroSource.fetch_us_cpi()`
- China CPI/PPI（MONTHLY_FIXED）：`AKShareMacroSource.fetch_china_cpi/ppi()`；
  备用 Tushare 宏观接口
- 联邦基金利率（EVENT_DRIVEN）：`FREDSource.fetch_series("DFEDTARL"/"DFEDTARU")`
- FOMC 日历（STATIC_YAML）：`fomc_calendar.yaml` → `MacroStorage.upsert_fomc()`
  （启动时加载一次）

**Group J — 大宗商品**：
- Brent (BZ=F) / WTI (CL=F) / XLU（INTRADAY_SLOW）：
  `YahooGlobalSource.fetch_ohlcv(["BZ=F", "CL=F", "XLU"])`

**Group K — 美国关键指数/权益**：
- SOX/COMP/NDX/SPX/DXY（INTRADAY_SLOW）：
  `YahooGlobalSource.fetch_ohlcv(["^SOX","^IXIC","^NDX","^GSPC","DX-Y.NYB"])`
- 10Y/2Y 国债收益率（DAILY）：
  `FREDSource.fetch_series("DGS10")` + `FREDSource.fetch_series("DGS2")`；
  10Y-2Y 利差在 storage upsert 后本地计算
- NVDA/MSFT/AVGO/ANET/VRT（INTRADAY_FAST）：
  `YahooGlobalSource.fetch_ohlcv([..."NVDA","MSFT","AVGO","ANET","VRT"...])`

## `scripts/refresh_macro_agent.py`

```bash
uv run refresh-macro-agent                        # 持续循环，60s poll
uv run refresh-macro-agent --once                 # 单次采集，退出
uv run refresh-macro-agent --once --groups I,K    # 限定 group 单次
uv run refresh-macro-agent --poll 120             # 自定义 poll 间隔
```

注册到 `pyproject.toml`（与 `refresh-data-agent` 并列）：


# Implementation Notes

- `run_once()` 中按 `UpdateClass` 的执行顺序建议：`INTRADAY_FAST` 优先，
  `STATIC_YAML`（FOMC YAML 加载）在启动初始化阶段执行，不进入主循环
- `pyproject.toml` 的 script 注册需验证 `uv run` 可正常找到入口函数
- `retrieval_log` 表由 `engine/data_agent/storage.py`（REQ-001/REQ-006）创建；
  本 REQ 直接写入已有表，不扩展 schema。macro 记录的 `field_group` 值为指标的
  `group_code`（"I"/"J"/"K" 等），`code` 字段填写 `indicator_id`
