# Goal

基于 DuckDB 中的筹码汇总数据，计算结构化分析信号，输出 `ChipAnalysis` Pydantic
model。信号逻辑封装在纯函数中，不依赖外部 API，便于测试和 LangGraph Agent 调用。

# In Scope

## `engine/agent/chip_analysis.py`

### Pydantic 模型

```python
from enum import Enum
from pydantic import BaseModel

class ChipLockLevel(str, Enum):
    HIGH   = "high"    # concentration >= 60
    MEDIUM = "medium"  # 40 <= concentration < 60
    LOW    = "low"     # concentration < 40

class ChipAnalysis(BaseModel):
    code: str
    date: str
    current_price: float

    # 原始汇总（来自 ChipSummary 透传）
    avg_cost: float
    profitable_pct: float      # 0–1
    concentration: float       # 0–100
    range_70_lower: float
    range_70_upper: float
    range_90_lower: float
    range_90_upper: float

    # 派生信号
    above_90_band: bool        # current_price > range_90_upper
    below_90_band: bool        # current_price < range_90_lower（深套区）
    cost_deviation_pct: float  # (current_price - avg_cost) / avg_cost * 100
    chip_spread_90: float      # range_90_upper - range_90_lower（筹码松散度）
    chip_lock_level: ChipLockLevel

    # 文字解读（不超过 80 字）
    signal_summary: str
```

### 信号计算规则

| 信号字段 | 计算逻辑 |
|---|---|
| `above_90_band` | `current_price > range_90_upper` |
| `below_90_band` | `current_price < range_90_lower` |
| `cost_deviation_pct` | `(current_price - avg_cost) / avg_cost * 100`，保留 1 位小数；`avg_cost ≤ 0` 时抛 `ValueError("Invalid avg_cost")` |
| `chip_spread_90` | `range_90_upper - range_90_lower` |
| `chip_lock_level` | `HIGH` if `concentration >= 60`；`MEDIUM` if `40 ≤ concentration < 60`；`LOW` if `< 40` |
| `signal_summary` | 见下方规则 |

### `signal_summary` 生成规则

纯字符串模板，不调用 LLM，根据信号组合拼装：

| 条件组合 | 摘要模板 |
|---|---|
| `above_90_band` and `profitable_pct >= 0.95` | `"价格突破90%筹码上界，{profitable_pct:.0%}筹码浮盈，获利兑现压力大"` |
| `above_90_band` and `profitable_pct < 0.95` | `"价格突破90%筹码上界，部分筹码仍在成本附近"` |
| `below_90_band` | `"价格低于90%筹码下界，处于历史套牢密集区"` |
| `not above_90_band and not below_90_band` and `cost_deviation_pct > 10` | `"价格高于均成本{cost_deviation_pct:.1f}%，筹码整体盈利"` |
| `not above_90_band and not below_90_band` and `cost_deviation_pct <= 0` | `"价格低于均成本，整体套牢状态"` |
| else | `"价格在90%筹码区间内，均成本偏离{cost_deviation_pct:.1f}%"` |

### 分析函数

```python
def analyze_chip(
    code: str,
    current_price: float,
    summary: ChipSummary,          # 来自 REQ-036 fetch_chip_summary
) -> ChipAnalysis:
    """纯函数：从 ChipSummary + 当前价格派生所有信号。无副作用，不访问数据库。"""

def load_and_analyze(
    code: str,
    current_price: float,
    conn: duckdb.DuckDBPyConnection,
    date: str | None = None,   # None → 取 chip_summary 表中该 code 的最新 date 行
) -> ChipAnalysis:
    """从 DuckDB 读取 ChipSummary（date=None 时取最新行），调用 analyze_chip 返回 ChipAnalysis。
    chip_summary 中无该 code 记录时抛出 ChipDataUnavailable(code=code)。"""
```

## Out of Scope

- LLM 生成分析文本（signal_summary 为纯模板，不调用 Claude）
- 多日筹码趋势对比（一期仅当日快照）
- 与 LangGraph ReAct Agent 的集成（Phase 12 扩展）

# Acceptance Criteria

- `TC-037-01`：`analyze_chip('688143', 131.60, mock_summary_688143)` 返回 `ChipAnalysis`，`above_90_band=True`（mock `range_90_upper=128.24`），`below_90_band=False`，`cost_deviation_pct` ≈ `+31.7`
- `TC-037-02`：`analyze_chip('000001', 12.0, mock_summary_below_band)` 返回 `below_90_band=True`，`signal_summary` 含 "套牢"
- `TC-037-03`：`ChipLockLevel` 枚举：`concentration=33.92` → `LOW`；`concentration=55.0` → `MEDIUM`；`concentration=65.0` → `HIGH`
- `TC-037-04`：`analyze_chip` 为纯函数 — 相同输入两次调用返回相同结果，无随机性
- `TC-037-05`：`load_and_analyze('688143', 131.60, conn)` 使用 in-memory DuckDB 预插入一行 chip_summary，返回 `ChipAnalysis`，`code='688143'`，`date` 与插入行一致
- `TC-037-06`：`load_and_analyze('000000', 10.0, conn)` 使用空 DuckDB（无该 code 记录）→ 抛出 `ChipDataUnavailable(code='000000')`
- `TC-037-07`：`analyze_chip` 接收 `avg_cost=0.0` 的 summary → 抛出 `ValueError("Invalid avg_cost")`

# Implementation Notes

- `analyze_chip` 不访问数据库或网络，便于单元测试全覆盖
- `signal_summary` 长度在测试中断言 `len(signal_summary) <= 80`
- 本 REQ 不包含 Streamlit UI（见 REQ-039）
- `ChipSummary` import 来自 `engine/agent/chip_fetcher`（REQ-036）
