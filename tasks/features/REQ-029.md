# Goal

防止 LLM 在 `signal_ranker` 中幻觉出与真实市场数据不符的数值声明（价格、涨跌幅、持仓量）。
`verify_signals()` 在信号写入 `AnalysisState` 之前对照 `FrozenMarketSnapshot` 做数值校验；
`prompt_contracts.py` 固化 System Prompt 并记录 SHA-256 hash，用于审计 prompt 漂移。

# In Scope

## `engine/agent/claim_verifier.py`

```python
from __future__ import annotations

from dataclasses import dataclass

from engine.agent.state import FrozenMarketSnapshot
from engine.schemas import StockSignal

_PRICE_TOLERANCE = 0.20  # allow 20% deviation (intraday movement since snapshot)


@dataclass
class ClaimViolation:
    code: str
    field: str
    claimed_value: str
    actual_value: str


def verify_signals(
    signals: list[StockSignal],
    snapshot: FrozenMarketSnapshot,
) -> list[ClaimViolation]:
    """Cross-check signal numeric claims against snapshot values.

    Returns empty list if all signals are clean.
    Violations do NOT block output — caller writes them to state["errors"].
    """
    violations: list[ClaimViolation] = []
    for sig in signals:
        actual_price = snapshot.price_snapshot.get(sig.code)
        if actual_price is not None and actual_price > 0:
            claimed = sig.signals.get("claimed_price")
            if claimed is not None:
                deviation = abs(float(claimed) - actual_price) / actual_price
                if deviation > _PRICE_TOLERANCE:
                    violations.append(
                        ClaimViolation(
                            code=sig.code,
                            field="price",
                            claimed_value=str(claimed),
                            actual_value=str(actual_price),
                        )
                    )
    return violations
```

## `engine/agent/prompt_contracts.py`

```python
from __future__ import annotations

from datetime import datetime
from hashlib import sha256

from engine.agent.nodes import _SIGNAL_RANKER_SYSTEM

SIGNAL_RANKER_SYSTEM_PROMPT: str = _SIGNAL_RANKER_SYSTEM
PROMPT_CONTRACT_HASH: str = sha256(SIGNAL_RANKER_SYSTEM_PROMPT.encode()).hexdigest()[:12]


def log_session_contract(session_id: str, db_path: str) -> None:
    """Append (session_id, PROMPT_CONTRACT_HASH, captured_at) to DuckDB for audit."""
    import duckdb

    con = duckdb.connect(db_path)
    try:
        con.execute(
            """
            CREATE TABLE IF NOT EXISTS prompt_audit (
                session_id   VARCHAR,
                prompt_hash  VARCHAR,
                captured_at  TIMESTAMP
            )
            """
        )
        con.execute(
            "INSERT INTO prompt_audit VALUES (?, ?, ?)",
            [session_id, PROMPT_CONTRACT_HASH, datetime.utcnow()],
        )
    finally:
        con.close()
```

## Integration with REQ-028 `signal_ranker`

在 `engine/agent/nodes.py` 的 `signal_ranker` 节点末尾（信号列表构建后）添加：

```python
from engine.agent.claim_verifier import verify_signals

violations = verify_signals(signals, snap)
for v in violations:
    errors.append(f"ClaimViolation[{v.code}][{v.field}]: claimed={v.claimed_value} actual={v.actual_value}")
```

# Out of Scope

- 价格 claim 以外的幻觉检测（持仓数量、市值等，延后）
- 阻断式校验（当前为 degraded mode：违规写入 errors，不过滤信号）
- A股涨跌停范围校验

# Acceptance Criteria

- `TC-029-01`：`verify_signals([sig_with_claimed_price_999], snap_with_price_100)`
  → 返回 1 个 `ClaimViolation`，`violation.field == "price"`，`violation.code` 匹配
- `TC-029-02`：`verify_signals([sig_with_no_claimed_price], snap)` → 返回空列表
  （signals 中没有 `claimed_price` key 时不报 violation）
- `TC-029-03`：`verify_signals([], snap)` → 返回空列表（无 signal 不报错）
- `TC-029-04`：`PROMPT_CONTRACT_HASH` 为长度 12 的非空字符串；重新 import
  `prompt_contracts` 后 hash 值不变（deterministic）

# Implementation Notes

- `_PRICE_TOLERANCE = 0.20`：允许快照和信号生成之间 20% 价格偏差，避免正常日内波动误报
- `claimed_price` 通过 `sig.signals` dict 传递（StockSignal 使用 `extra="allow"`），
  LLM 需在 system prompt 中指示输出此字段以触发验证
- 当前版本仅验证 `claimed_price`——未来可扩展到 `claimed_revenue_growth` 等字段
- `PROMPT_CONTRACT_HASH` 变更时，DuckDB 审计表中的历史记录可对比，追溯 prompt 更新时间点
