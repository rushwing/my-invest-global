# Goal

用 Claude Vision API 解析同花顺 App 截图，从截图中提取筹码分布指标，返回与
akshare 路径格式一致的 `ChipAnalysis`。作为 akshare 数据不可用时的备用输入路径，
也支持用户手动导入历史截图。

## 适用截图类型

根据测试（2026-05-19 长盈通 688143）：

| 截图类型 | 信息密度 | 是否适合 OCR |
|---|---|---|
| **PC K 线 + 筹码分布直方图**（图4） | 最高：含直方图形状 + 收盘获利/平均成本/90%区间/集中度 | ✅ 优先使用 |
| 手机资金流向（图5） | 含主力流向/大单分解，**不含**筹码分布字段 | ❌ 不适用 |
| 手机 K 线基本面（图6） | 含换手率等，**不含**筹码分布字段 | ❌ 不适用 |

**结论：仅接受含筹码分布直方图的截图（同花顺 PC 端或手机端「筹码分布」Tab）。**

# In Scope

## `engine/agent/chip_screenshot_parser.py`

### 接口

```python
def parse_chip_screenshot(
    image_path: str,           # 本地文件路径，支持 PNG/JPG
    code: str | None = None,   # 已知股票代码（用于输出 ChipAnalysis.code）；None 时从截图提取
) -> ChipAnalysis:
    """
    调用 Claude Vision API（claude-sonnet-4-6）解析同花顺筹码截图。
    从截图中提取：code, avg_cost, profitable_pct, concentration,
    range_70/90 bounds，current_price。
    调用 analyze_chip() 派生信号字段后返回完整 ChipAnalysis。
    截图不含筹码分布信息时抛出 ScreenshotParseError。
    """
```

### Prompt 策略

系统提示要求 Claude 以 JSON 格式返回以下字段（无法识别时填 `null`）：

```json
{
  "code": "688143",
  "current_price": 131.60,
  "avg_cost": 99.96,
  "profitable_pct": 0.9973,
  "concentration": 33.92,
  "range_70_lower": null,
  "range_70_upper": null,
  "range_90_lower": 63.28,
  "range_90_upper": 128.24
}
```

返回后用 Pydantic 校验（字段值范围断言）：
- `0 < avg_cost < 100000`
- `0 ≤ profitable_pct ≤ 1`
- `0 ≤ concentration ≤ 100`
- `range_90_lower < range_90_upper`

任何 `null` 关键字段（`avg_cost`/`profitable_pct`/`range_90_lower`/`range_90_upper`）→ 抛出 `ScreenshotParseError`。

**range_70 fallback 规则**：`range_70_lower`/`range_70_upper` 为 `null` 时（图4截图中70%区间通常无明确数值），
使用 `range_90_lower`/`range_90_upper` 作为 fallback，不抛异常。
这样传入 `analyze_chip()` 的 `ChipSummary` 中 range_70 字段始终为 `float`，
无需改动 `ChipSummary` / `ChipAnalysis` 的类型定义（保持 `float`，不改为 `Optional`）。

### 异常

```python
class ScreenshotParseError(Exception):
    def __init__(self, reason: str):
        ...
```

### Claude API 调用规范

- 模型：`claude-sonnet-4-6`
- 图像编码：base64，`media_type` 从文件扩展名推断
- `max_tokens`: 256（JSON 输出紧凑）
- 禁止 prompt caching（截图内容每次不同）
- 调用失败（网络/API 错误）→ 抛出 `ScreenshotParseError(reason="API call failed: ...")`

## 融合逻辑（供调用方参考）

```python
# 推荐用法：优先 akshare，截图作 fallback
try:
    summary = fetch_chip_summary(code)
    analysis = analyze_chip(code, current_price, summary)
except ChipDataUnavailable:
    analysis = parse_chip_screenshot(image_path, code=code)
```

## Out of Scope

- 批量截图解析
- 手机端资金流向截图（字段不匹配，不做 OCR）
- 自动截图采集（需用户手动提供图片路径）
- 筹码直方图形状的像素级解析（仅解析文字字段）
