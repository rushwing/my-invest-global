## Summary

在 Streamlit 会话层实现单用户本地口令认证。未解锁时主内容区显示模糊遮罩；正确口令经 bcrypt 验证后解锁，会话内持久。

## Design Reference

`docs/design/dashboard/design_handoff_dashboard/holdings-editor-auth.jsx`：
- Artboard A：收起态 + 已锁定（灰色头像 + 锁图标）
- Artboard B：展开态 + 登录表单（口令输入框 + 解锁按钮）
- Artboard C：展开态 + 错误状态（红色提示"口令错误"）
- Artboard D：展开态 + 已解锁（蓝色头像 + 绿点）
- Artboard E：收起态 + 已解锁（蓝色头像 + 绿点）

## Acceptance Criteria

- 输入错误口令 → `st.session_state["unlocked"]` 保持 `False`，显示红色错误提示
- 输入正确口令 → `st.session_state["unlocked"]` 设为 `True`，主区域遮罩消失，`st.rerun()`
- 点击 Header 锁定按钮 → 清除 `session_state["unlocked"]` → `st.rerun()`，返回登录态
- 刷新页面（新会话）→ `session_state["unlocked"]` 初始化为 `False`，必须重新输入口令
- `.env` 中 `DASHBOARD_PASSPHRASE_HASH` 缺失 → 启动时 `st.error()` 提示配置错误，不崩溃

## Implementation Details

### 配置

```env
# .env
DASHBOARD_PASSPHRASE_HASH=<bcrypt hash of passphrase>
```

生成命令（首次配置）：
```python
import bcrypt
print(bcrypt.hashpw(b"your-passphrase", bcrypt.gensalt()).decode())
```

### 核心逻辑

```python
# app/auth.py
import os, bcrypt, streamlit as st

def is_unlocked() -> bool:
    return st.session_state.get("unlocked", False)

def try_unlock(passphrase: str) -> bool:
    stored = os.getenv("DASHBOARD_PASSPHRASE_HASH", "").encode()
    if stored and bcrypt.checkpw(passphrase.encode(), stored):
        st.session_state["unlocked"] = True
        return True
    return False

def lock():
    st.session_state.pop("unlocked", None)
    st.rerun()
```
