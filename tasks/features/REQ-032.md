# Goal

`HoldingRow` 增加 `account: str` 字段，标识持仓所在券商账户（华西证券 / 西南证券），提升数据准确性和可追溯性。在双账户持有同一标的（如 603986 兆易创新）时，聚合行能正确标注账户来源。

# In Scope

## `engine/schemas.py` — HoldingRow 新增字段

```python
class HoldingRow(BaseModel):
    # 现有字段不变 ...
    account: str = ""   # 券商账户名，如 "华西证券" / "西南证券"；空字符串向后兼容旧 YAML
```

`extra="forbid"` 需改为允许新字段加载（或直接加字段）。

## `data/agent_input/cn/holdings.yaml` — 每条加 account

账户分配：
- 华西证券 **8175：601989 中国船舶、603986 兆易创新（部分）、002384 东山精密
- 西南证券 **5426：300308 中际旭创、300394 天孚通信、603986 兆易创新（部分）、688820 盛合晶微、002957 科瑞技术

每条 holding 加一行 `account: "华西证券"` 或 `account: "西南证券"`。

## `engine/portfolio.py` — _aggregate_holdings 处理 account

合并同 code 的持仓时：
- 若各行 account 相同 → 保留原值
- 若不同 → `"+".join(sorted(set(accounts)))`，例如 `"华西证券+西南证券"`

## `app/dashboard.py` — Holdings 表格加账户列

在持仓行的 dict 中加入 `"账户": h.account`，使持仓表格显示账户列。
