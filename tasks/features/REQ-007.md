# Goal

实现 `engine/macro_agent/storage.py`（`MacroStorage` 类），在共享 DuckDB
文件（`data/db/aidc.duckdb`）中创建并管理宏观指标所需的 4 张新表，
提供类型安全的 upsert 辅助方法。

# In Scope

## `engine/macro_agent/storage.py`

```python
class MacroStorage:
    def __init__(self, db_path: str | Path | None = None) -> None: ...
    # 内部使用与 engine/data_agent/storage.py 相同的 _find_project_root() 工具

    def upsert_indicators(self, records: list[dict]) -> int: ...
    # 批量 upsert macro_indicators；返回写入行数

    def upsert_capex(self, records: list[dict]) -> int: ...
    # 批量 upsert capex_quarterly

    def upsert_fomc(self, records: list[dict]) -> int: ...
    # upsert fomc_calendar（通常逐行）

    def upsert_regime(self, record: dict) -> None: ...
    # upsert macro_regime（按 as_of_date）

    def get_latest_indicator(self, indicator_id: str, source: str | None = None) -> dict | None: ...
    # 按 fetched_at DESC 取最新一条；source 非 None 时追加 WHERE source=source

    def get_capex_quarters(self, company: str, n: int = 4) -> list[dict]: ...
    # 按 period_date DESC 取最近 n 个季度，返回列表顺序为最新在前

    def get_fomc_upcoming(self, from_date: date, lookahead_days: int = 90) -> list[dict]: ...
    # WHERE meeting_date >= from_date AND meeting_date <= from_date + lookahead_days（两端包含）
    # 按 meeting_date ASC 排序

    def get_regime_latest(self) -> dict | None: ...
    # 返回最新 macro_regime 行

    def get_av_budget(self, date: date) -> int: ...
    # 返回当日 Alpha Vantage 已使用请求数

    def increment_av_budget(self, date: date) -> None: ...
    # 当日请求数 +1
```

## DuckDB 表 DDL（参照 ADR-007 决策四）

4 张表（见 ADR-007 目录结构节）+ 1 张预算台账：

```sql
CREATE TABLE IF NOT EXISTS alpha_vantage_budget (
    date            DATE        PRIMARY KEY,
    requests_used   INT         NOT NULL DEFAULT 0
);
```

Upsert 模式：`INSERT INTO ... ON CONFLICT (...) DO UPDATE SET ...`（与 `data_agent/storage.py` 一致）。

# Acceptance Criteria

- `TC-007-01 round-trip upsert`：插入 `macro_indicators` 一行，再 upsert 相同
  PK + 更新 `value`，验证存储值为新值（ON CONFLICT DO UPDATE）
- `TC-007-02 vintage 保留`：upsert 含 `realtime_start="2026-04-01"` 的 FRED
  记录后，`get_latest_indicator()` 按 `fetched_at DESC` 返回最新行且 `realtime_start` 字段不丢失
- `TC-007-03 capex 反算值保留`：upsert 4 条 `capex_quarterly` 记录后，
  `get_capex_quarters("MSFT", 4)` 返回 4 条且第 0 项为 period_date 最新的季度，`capex_usd` 值正确
- `TC-007-04 AV 预算台账`：
  (a) 无记录时 `get_av_budget(date)` 返回 0；
  (b) 连续调用 `increment_av_budget(date)` 22 次后，`get_av_budget(date)` 返回 22
- `TC-007-05 FOMC 查询`：预填充 3 条 `fomc_calendar` 记录（1 条在 from_date 前、
  1 条在窗口内、1 条超出 lookahead_days），`get_fomc_upcoming(from_date, 90)` 只返回
  窗口内 1 条，且 meeting_date >= from_date，meeting_date <= from_date + 90d（两端包含）
- `TC-007-06 regime round-trip`：`upsert_regime` 写入一条 `macro_regime` 记录，
  `get_regime_latest()` 返回相同 `as_of_date` 和全部字段；再 upsert 相同 PK + 更新
  字段值，验证 `get_regime_latest()` 返回新值

# Implementation Notes

- `db_path: str | Path | None`：`None` 时通过 `_find_project_root()` 定位
  `data/db/aidc.duckdb`；传入 `":memory:"` 时直接使用 DuckDB in-memory 模式，
  **不创建 parent 目录**（测试用）；传入具体路径时确保 parent 目录存在再连接
- `_INIT_SQL` 包含全部 5 张表的 `CREATE TABLE IF NOT EXISTS` DDL；在
  `__init__` 中执行，保证幂等
- 与 `data_agent/storage.py` 共享同一个 DuckDB 文件（`aidc.duckdb`），
  但 `MacroStorage` 管理独立的连接实例（DuckDB 支持多连接到同一文件）
- `increment_av_budget(date)`：无记录时 INSERT `requests_used=1`；有记录时
  `ON CONFLICT (date) DO UPDATE SET requests_used = requests_used + 1`
- `get_av_budget(date)`：无记录时返回 `0`（不抛异常）
