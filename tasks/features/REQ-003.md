# Goal

定义 `SignalProvider` Protocol 接口，使 `run_daily_pipeline()` 能通过依赖注入接受任意符合协议的信号计算实现，同时提供 `StubSignalProvider` 用于 Phase 5 集成测试（Phase 2–4 的真实实现后续注入，无需修改 runner）。

# In Scope

- `engine/signals/__init__.py`（新建，空文件，标记为 package）
- `engine/signals/protocol.py`：
  - `@runtime_checkable class SignalProvider(Protocol):`
  - 方法：`def compute_score(self, code: str, date: str) -> float: ...`
  - 返回值范围约定（非强制验证）：`0.0 ≤ score ≤ 100.0`
  - 类型注解完整，`mypy --strict` 通过
- `engine/signals/stub.py`：
  - `class StubSignalProvider:`（不显式继承 Protocol，通过结构化子类型自动匹配）
  - `def __init__(self, base_score: float = 70.0) -> None`
  - `def compute_score(self, code: str, date: str) -> float:`：始终返回 `self.base_score`，忽略参数
  - 用于测试，不用于生产

# Out of Scope

- `engine/signals/technical.py`（Phase 2 实现）
- `engine/signals/fundamental.py`（Phase 3 实现）
- `engine/signals/sentiment.py`（Phase 4 实现）
- 任何真实市场数据访问

# Acceptance Criteria

- `from engine.signals.protocol import SignalProvider` 可正常 import
- `from engine.signals.stub import StubSignalProvider` 可正常 import
- `isinstance(StubSignalProvider(), SignalProvider)` 返回 `True`（`runtime_checkable` 有效）
- `StubSignalProvider(base_score=85.0).compute_score('300308', '2026-05-13')` 返回 `85.0`
- `StubSignalProvider()` 无参构造，`.compute_score('600036', '2026-05-13')` 返回 `70.0`（默认值）
- `uv run mypy engine/signals/ --strict` 通过

# Implementation Notes

- `SignalProvider` 使用 `typing.Protocol`（Python 3.8+），需 `from typing import runtime_checkable, Protocol`
- 不引入任何第三方依赖
- `StubSignalProvider` 是最简实现，仅用于测试隔离；不要添加任何"真实"逻辑（如随机数、噪声等）
- `tc_policy: optional`：本 REQ 为纯接口定义，无行为逻辑，不单独编写 TC 文件；但 REQ-004 的 TC 将通过 `StubSignalProvider` 隐式验证此接口
