# Goal

实现 `engine/shared/sources/` 下的 6 个宏观数据源基础模块，为 PHASE-008
后续 REQ 提供可复用的 source 层。所有 source 继承
`engine/data_agent/sources/base.py` 的 `AbstractSource`。

# In Scope

## `engine/shared/sources/base_macro.py`

`MacroAbstractSource(AbstractSource)` — 在 `AbstractSource` 基础上添加：
- `_to_utc_ts(local_dt, tz_str) -> datetime`：本地时间转 UTC
- `_make_period_record(indicator_id, group_code, period_date, market_tz, value, ...)
  -> dict`：构造符合 `macro_indicators` 表结构的记录字典

## `engine/shared/sources/fred.py`

`FREDSource(MacroAbstractSource)`：
- `name = "fred"`, `domain = "api.stlouisfed.org"`
- `fetch_series(series_id, start_date, realtime_start=None) -> list[dict]`：
  调用 FRED `/fred/series/observations`，支持 `realtime_start` 参数（BP-1）
- API key 从环境变量 `FRED_API_KEY` 读取；缺失时 raise `RuntimeError`
- 返回记录包含 `realtime_start`, `vintage_as_of`（取 fetch 当日）字段

## `engine/shared/sources/sec_edgar.py`

`SECEdgarSource(MacroAbstractSource)`：
- `name = "sec_edgar"`, `domain = "data.sec.gov"`
- User-Agent 强制注入（BP-2）：`"my-invest-global ruoxu.wang@gmail.com"`，
  在 `__init__` 中覆盖 session header，不依赖调用方设置
- 速率限制器 domain 配置上限 8 req/s（在 macro_agent 的 `DOMAIN_CONFIGS` 中）
- `fetch_capex_quarterly(cik, tag="PaymentsToAcquirePropertyPlantAndEquipment")
  -> list[dict]`：
  - 请求 `https://data.sec.gov/api/xbrl/companyfacts/CIK{cik}.json`
    （`cik` 须保留前导零，如 `CIK0000789019`；直接用裸数字会 404）
  - 提取指定 XBRL tag 的季度值
  - 执行单季反算（BP-3）：
    - 10-Q：`Q_single = Q_cumulative - prev_Q_cumulative`（同财年前一季累计值）
    - 10-K（Q4）：`Q4_single = FY_annual - (Q1 + Q2 + Q3 单季值)`
      （不能直接用年度值当 Q4，否则 QoQ/YoY 门控严重失真）
    - Q1（无前置季）直接使用累计值作为单季值
  - 计算 SHA-256 哈希并写入 `source_hash`（BP-10）
  - 返回字段：`company`, `cik`, `fiscal_quarter`（格式 "2026Q1"）,
    `period_end`, `capex_usd`（单季正值）, `filing_form`, `source_hash`

## `engine/shared/sources/yahoo_global.py`

`YahooGlobalSource(MacroAbstractSource)`：
- `name = "yahoo_global"`, `domain = "query1.finance.yahoo.com"`
- `_refresh_crumb()` 方法：从 `https://finance.yahoo.com` 获取 crumb token，
  TTL 24h，懒加载（BP-8）
- `fetch_ohlcv(tickers, period="1d") -> list[dict]`：批量拉取日线 OHLCV，
  内部调用 yfinance；结果含 `period_date`, `market_tz`, `utc_ts`（BP-5）
- `fetch_quote_summary(ticker, modules=["cashflowStatementHistoryQuarterly"])
  -> dict`：调用 `quoteSummary` 端点（需 crumb），用于 CapEx Yahoo 备源

## `engine/shared/sources/alpha_vantage.py`

`AlphaVantageSource(MacroAbstractSource)`：
- `name = "alpha_vantage"`, `domain = "www.alphavantage.co"`
- API key 从环境变量 `ALPHA_VANTAGE_KEY` 读取
- `fetch_news_sentiment(tickers=None, topics=None, limit=50) -> list[dict]`：
  调用 `NEWS_SENTIMENT` 函数端点
- 每次调用前检查 DuckDB `alpha_vantage_budget` 表（BP-9）：
  `requests_used >= 22` 时 raise `BudgetExhaustedError`（自定义异常），
  不发送实际请求；调用成功后 +1 计数

## `engine/shared/sources/akshare_macro.py`

# Implementation Notes

- `base_macro.py` 中 `_make_period_record()` 生成的 dict key 必须与
  `REQ-007` 的 `macro_indicators` 表列名完全对应，避免 upsert 时列名不匹配
- `SECEdgarSource` 的单季反算需要处理边界情况：Q1（无前一季）直接使用累计值；
  同财年同 tag 值不按时间顺序排列时需先排序
- `YahooGlobalSource` 中 crumb 失效（401）时自动刷新一次，再失败则 raise `SourceError`
