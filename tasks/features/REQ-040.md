新增 KPI 进度条卡片、Plotly 仓位甜甜圈 + PnL 热力地图、内联 `st.data_editor` CRUD 表格。

## Design Reference

`docs/design/dashboard/design_handoff_dashboard/tab1-holdings-v2.jsx`：
- `KpiRow`：3 卡片（总市值 + 浮盈、弹性仓位 + 进度条 + 目标标记线、白马仓位 + 进度条）
- `DonutMini`：Plotly Pie，`hole=0.62`，内圈标注总市值

- `HoldingsTableV2`：`st.data_editor`，分类筛选器，脏状态检测，保存快照，CSV 导入

## Acceptance Criteria

- KPI 卡片：总市值含浮盈/浮亏 delta；弹性/白马仓位含进度条 + 目标竖线；偏差颜色：|dev| ≤5% → bull，5–10% → neutral，>10% → bear
- Plotly 甜甜圈按类别着色（白马 #4F8EF7 / 弹性 #00C47A / 自选 #F5A623）
- Plotly Treemap 颜色 scale：-20% → `#3D1515`，0 → `#1C1C2E`，+20% → `#0D3320`
- `st.data_editor`：代码/名称/类别/成本价/现价/数量 可编辑；市值/浮盈% 计算列只读
- 分类筛选器（全部/白马股/弹性股/自选）：切换时先把编辑态同步回全量 df，再显示筛选子集
- `保存快照`：脏状态时显示"● 保存"（primary 样式）；空 code 行不写入 CSV
- CSV 导入：`dtype={"code": str}` + `zfill(6)` 归一化；逐行追加，不按 code 去重（多批次同 code 合法）
- 导入/初始化时 NaN/空 code 归一化为 `None`，不写入字符串 `"nan"`
- 侧边栏保存后清空 `t1_holdings_full` session state，Tab 1 下次渲染重新从 CSV 加载
- Tab 3 跳转快捷入口：设置 `session_state["tab3_code"]` 后提示切换标签
