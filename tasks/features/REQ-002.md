# Goal

将现有 `engine/portfolio.py`（单文件模块）重构为 `engine/portfolio/` 包，保持所有公共 API 的 import 路径不变，并新增 `check_position_too_large()` 函数检测单只弹性股持仓过大。

# In Scope

- **重构（不破坏现有 API）**：
  - 将 `engine/portfolio.py` 重命名 / 移动为 `engine/portfolio/__init__.py`
  - `engine/portfolio/__init__.py` 内容与原 `engine/portfolio.py` 完全相同（或重导出，见 Implementation Notes）
  - `from engine.portfolio import check_portfolio_balance, TARGET_ELASTIC_BY_MACRO_STATE` 继续有效

- **新文件 `engine/portfolio/framework.py`**：
  - 从 `engine/portfolio/__init__.py` 重导出 `check_portfolio_balance`
  - 新增 `check_position_too_large(holdings_df: pd.DataFrame, threshold: float = 0.10) -> list[str]`：
    - 过滤 `holding_category == 'elastic'` 的行
    - 计算每行 `market_value / total_market_value`
    - 返回比例严格大于 `threshold` 的股票代码列表（空列表表示无超限）
    - 所有代码使用字符串类型（如 `"300308"`）

- **新测试 `tests/test_framework.py`**：见 § Test Cases

# Out of Scope

- `macro_gate.py`（REQ-001 负责）
- runner 集成（REQ-004 负责）
- 修改 `check_portfolio_balance()` 的任何逻辑（只移动，不改）

# Acceptance Criteria

- `from engine.portfolio import check_portfolio_balance, TARGET_ELASTIC_BY_MACRO_STATE` 不报 ImportError
- `from engine.portfolio.framework import check_portfolio_balance, check_position_too_large` 均有效
- `uv run pytest tests/test_portfolio.py -v` 全部通过（不修改该测试文件）
- `check_position_too_large(df)` 对弹性股持仓比例恰好等于10%时返回空列表（边界：不触发）
- `check_position_too_large(df)` 对弹性股持仓比例为10.01%时返回含该股代码的列表
- `check_position_too_large(df)` 对多只弹性股均超限时返回包含所有超限代码的列表
- `uv run mypy engine/portfolio/ --strict` 通过

# Test Cases

需覆盖以下场景（TC文件由 evaluator-001 在 `tc_design` 阶段编写）：

1. **TC-002-01 边界值：恰好10%不触发**：一只弹性股市值/总市值 = 0.10 → 返回 `[]`
2. **TC-002-02 边界值：10.01%触发**：一只弹性股市值/总市值 = 0.1001 → 返回 `['<code>']`
3. **TC-002-03 多只超限**：两只弹性股均超10% → 返回两个代码（顺序不限）
4. **TC-002-04 白马股不计入**：白马股占40%，弹性股均不超10% → 返回 `[]`
5. **TC-002-05 无弹性股**：所有持仓均为白马股 → 返回 `[]`
6. **TC-002-06 import 路径兼容性**：`from engine.portfolio import check_portfolio_balance` 路径有效

# Implementation Notes

- `engine/portfolio/__init__.py` 可选方案：
  - **方案A（推荐）**：将原 `engine/portfolio.py` 内容原封不动移入 `__init__.py`
  - **方案B**：`__init__.py` 中 `from engine.portfolio.framework import *`（需确保所有公共符号可用）
  - 选择方案A以最小化变更风险，`tests/test_portfolio.py` 无需任何修改
- `check_position_too_large()` 的 `holdings_df` 需包含列：`code`（str）、`market_value`（float）、`category`（str，值为 `'white_horse'` 或 `'elastic'`）
  - `total_market_value` 从 `holdings_df['market_value'].sum()` 计算，不是外部参数
- 函数参数命名遵循 GLOSSARY：`holding_category` 字段值为 `'white_horse'` / `'elastic'`（与 `HoldingCategory` enum 一致）
- REQ-001 在此 REQ 之前完成，因此 `engine/portfolio/__init__.py` 已存在（REQ-001 创建了空文件）；本 REQ 将实际内容写入
