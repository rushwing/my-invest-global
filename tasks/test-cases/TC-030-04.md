## Objective

验证当 `analysis_sessions` 表为空时，`list_sessions(":memory:", limit=30)` 返回空列表 `[]` 而不抛异常。

## Preconditions

- `engine.agent.session_store.list_sessions` 可 import
- DuckDB `:memory:` 全新连接（空表）

## Steps

1. 打开 `:memory:` DuckDB（不调用 `save_session`）
2. 调用 `rows = list_sessions(":memory:", limit=30)`

## Expected Result

- `rows` == `[]`（空列表，不是 `None`）
- 不抛任何异常
- 表在查询过程中被自动创建（`CREATE TABLE IF NOT EXISTS`）

## Notes

- 返回值类型一致性：空表返回 `[]` 而非 `None`，方便调用方直接 `for row in rows` 遍历
- 与 `load_latest_session` 返回 `None` 不同——`list_sessions` 始终返回 `list`
