## Objective

验证 `signal_ranker` 在 `technical_data` 非空时，构建的 `source_index[code]` 包含 T1..T8 数据源引用，且每个 T-ref 的 content 包含对应指标名和数值。

## Preconditions

- `engine.agent.nodes.signal_ranker` 可 import（mock LLM 注入）
- 构造含 `technical_data` 的 AnalysisState（一只股票，含 MA5/MACD_DIF 等全部 8 个键）
- mock LLM 返回合法 JSON，包含该股票的基本字段

## Steps

1. 构造 AnalysisState，technical_data 含 300308 的完整指标：
   ```python
   state["technical_data"] = {
       "300308": {
           "MA5": 1045.2, "MA10": 1038.5, "MA20": 1020.0, "MA30": 1005.0,
           "MACD_DIF": 12.34, "MACD_DEA": 10.21, "MACD_BAR": 4.26, "RSI14": 62.1,
       }
   }
   ```
2. 注入 mock LLM，调用 `result = signal_ranker(state, llm=mock_llm)`
3. 验证 source_index：
   ```python
   idx = result["source_index"]["300308"]
   assert "T1" in idx and "MA5" in idx["T1"]
   assert "T5" in idx and "MACD DIF" in idx["T5"]
   assert "T8" in idx and "RSI" in idx["T8"]
   ```

## Expected Result

- `source_index["300308"]` 包含 T1..T8 全部 8 个键
- T1 content 含 `"MA5"` 字样
- T5 content 含 `"MACD DIF"` 字样
- T8 content 含 `"RSI"` 字样

## Notes

- 当 technical_data 中某个指标为 NaN 时，对应 T-ref 应被跳过（不出现在 source_index）
- mock LLM 输出只需基本字段，不需要含 T-ref 引用（测试的是 source_index 构建，不是 Claude 输出）
