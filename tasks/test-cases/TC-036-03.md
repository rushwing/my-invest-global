## Objective

验证 `store_chip_data` 对同一 `(code, date)` 重复写入不报错，并且不会产生重复 summary 行。

## Preconditions

- `store_chip_data` 可 import
- in-memory DuckDB 已通过 `init_db(conn)` 初始化
- 准备两个同 code/date 但数值不同的 `ChipSummary`

## Steps

1. 写入第一份 summary。
2. 写入第二份同 `code/date` 的 summary，调整 `avg_cost` 或 `profitable_pct`。
3. 查询：
   ```sql
   SELECT COUNT(*) FROM chip_summary WHERE code='688143' AND date='2026-05-19';
   SELECT avg_cost FROM chip_summary WHERE code='688143' AND date='2026-05-19';
   ```
4. 断言 count 为 1。
5. 断言最终行是第二次写入值，或至少满足实现声明的 deterministic UPSERT 语义。

## Expected Result

- 二次写入不抛异常
- `chip_summary` 对同一 `(code, date)` 不出现重复行
- `chip_distribution` 对同一 `(code, date, price_lower)` 不出现重复行

## Notes

- 若 DuckDB 版本不支持原生 `ON CONFLICT`，实现可用 delete-then-insert 达到同等语义
