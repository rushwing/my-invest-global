## Objective

验证 `fetch_capex_quarterly()` 对 10-K 年度值执行 Q4 = FY - Q3_cum 反算，
不直接将年度值当作 Q4 单季值。

## Fixture (fy=2025, 单位 USD)

```json
{
  "facts": {
    "us-gaap": {
      "PaymentsToAcquirePropertyPlantAndEquipment": {
        "units": {
          "USD": [
            {"end": "2025-03-31", "val": 3000000000, "form": "10-Q", "fp": "Q1", "fy": 2025},
            {"end": "2025-06-30", "val": 7000000000, "form": "10-Q", "fp": "Q2", "fy": 2025},
            {"end": "2025-09-30", "val": 10500000000, "form": "10-Q", "fp": "Q3", "fy": 2025},
            {"end": "2025-12-31", "val": 14000000000, "form": "10-K", "fp": "FY", "fy": 2025}
          ]
        }
      }
    }
  }
}
```

## Steps

1. Mock `_get` 返回上述 fixture
2. `records = src.fetch_capex_quarterly("0000789019")`
3. 取 `fp == "Q4"`（或 `fiscal_quarter` 含 "Q4"）的记录

## Expected Result

- `q4_record["capex_usd"]` == `3.5e9`
  （FY 14B - Q3_cum 10.5B = 3.5B；若直接用 FY 值则为 14B，错误）
- `q4_record["filing_form"]` == `"10-K"`

## Rationale

错误实现会直接将 FY=14B 存为 Q4_single，导致 QoQ/YoY 门控失真约 4×。
