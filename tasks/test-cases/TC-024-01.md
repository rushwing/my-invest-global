## Objective

验证 mock LLM 生成只读 `MATCH ... RETURN` Cypher 时，`KGQuerier.query()` 会执行注入的
`session_runner` 并返回结构化结果。

## Preconditions

- 注入 mock `llm`
- 注入 schema 字符串，避免真实 Neo4j schema 读取
- 注入 `session_runner`，返回 `[{"s.code": "300308"}]`
- mock LLM 第一次返回 Cypher，第二次返回中文摘要 `"中际旭创属于光通信"`

## Steps

1. 调用 `KGQuerier(conn).query(question, llm=mock_llm, schema=schema, session_runner=runner)`。
2. 检查返回的 `KGQueryResult`。

## Expected Result

- `result.cypher_used` 包含 `"MATCH"`
- `result.raw_results == [{"s.code": "300308"}]`
- `result.answer == "中际旭创属于光通信"`
- `result.error is None`
- `session_runner` 被调用 1 次
