## Objective

验证宏观 agent 在全新 DuckDB 或 `:memory:` 数据库中不依赖 data_agent
`Storage` 初始化，也能写入共享 `retrieval_log`。

## Preconditions

- 使用 `MacroStorage(":memory:")` 构造 orchestrator
- 所有 source 使用 mock
- scheduler 返回两个 due 指标：一个成功、一个所有 source 均失败

## Steps

1. 构造 `MacroOrchestrator` 并注入 `MacroStorage(":memory:")`。
2. 调用 `run_once()`。
3. 查询 `retrieval_log` 表。

## Expected Result

- `run_once()` 不抛出 “table retrieval_log does not exist”
- `retrieval_log` 表存在
- 至少一条记录 `status == "ok"`，包含 `source` 和 `latency_ms`
- 至少一条记录 `status == "failed"`，包含 `error_msg`
- macro 记录中 `field_group` 为 group code，`code` 为 `indicator_id`
