## Objective

验证 BLS 发布日历抓取失败时安全降级，不抛异常且不清空已有数据。

## Preconditions

- 使用 `MacroStorage(":memory:")`
- `release_dates` 中已有一条 CPIAUCSL 发布日记录
- HTTP 请求被 mock 为返回 500 或抛出网络异常

## Steps

1. 构造 `ReleaseCalendar(storage)`。
2. 调用 `populate_bls(2026)`。
3. 查询 `release_dates` 中 CPIAUCSL 的原有记录。

## Expected Result

- `populate_bls()` 不抛异常
- 日志中出现 warning
- 原有 `release_dates` 记录仍存在且未被删除
