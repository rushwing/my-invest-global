## Objective

验证 `technical_fetcher` LangGraph 节点在 akshare 抛出异常时，不中断 pipeline，而是返回空 `technical_data` 并向 `errors` 追加警告。

## Preconditions

- `engine.agent.nodes.technical_fetcher` 可 import
- `engine.agent.state.AnalysisState` 可构造（最小 mock snapshot）
- `unittest.mock.patch` 可用于 mock `engine.agent.technical_fetcher.fetch_technicals`

## Steps

1. 构造最小 AnalysisState（含 mock snapshot 和 errors=[]）
2. patch `fetch_technicals` 抛出 `ConnectionError("mock network error")`
3. 调用 `result = technical_fetcher(state)`
4. 验证：
   ```python
   assert result["technical_data"] == {}
   assert len(result["errors"]) == 1
   assert "technical_fetcher" in result["errors"][0]
   ```

## Expected Result

- `technical_data` 为空 dict（不是 None）
- `errors` 列表包含一条以 "technical_fetcher" 开头的错误信息
- 节点正常返回（无异常抛出）

## Notes

- 正常 run_analysis() 流程在 technical_data={} 情况下应继续运行并生成 signals（T-refs 缺失但不影响 Claude 输出）
- 测试对象是节点函数本身，不需要 LangGraph StateGraph 编译
