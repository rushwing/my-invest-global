## Objective

验证每条 `capex_quarterly` 记录的 `source_hash` 字段是对原始 JSON payload
的 SHA-256 hex digest（64 位小写十六进制字符串）。

## Steps

1. Mock `_get` 返回含 Q1 记录的最小 fixture
2. `records = src.fetch_capex_quarterly("0000789019")`
3. 取任一记录的 `source_hash` 字段

## Expected Result

- `len(source_hash) == 64`
- `source_hash.islower() and all(c in "0123456789abcdef" for c in source_hash)`
- 同一 fixture 的两次调用返回相同 `source_hash`（确定性）
