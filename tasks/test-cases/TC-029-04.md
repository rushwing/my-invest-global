## Objective

验证 `PROMPT_CONTRACT_HASH` 是对 `_SIGNAL_RANKER_SYSTEM` 稳定的 SHA-256 前 12 位 hex digest，且跨 import 不变。

## Preconditions

- `engine.agent.prompt_contracts` 可 import
- `engine.agent.nodes._SIGNAL_RANKER_SYSTEM` 可 import

## Steps

1. `from engine.agent.prompt_contracts import PROMPT_CONTRACT_HASH`
2. 记录 `hash1 = PROMPT_CONTRACT_HASH`
3. 通过 `importlib.reload()` 或重新 `import` 模块，再次读取 `hash2 = PROMPT_CONTRACT_HASH`
4. 直接计算 `expected = sha256(_SIGNAL_RANKER_SYSTEM.encode()).hexdigest()[:12]`

## Expected Result

- `len(PROMPT_CONTRACT_HASH)` == 12
- `PROMPT_CONTRACT_HASH` 为非空字符串（仅含 hex 字符 0-9 a-f）
- `hash1 == hash2`（re-import 后值不变）
- `hash1 == expected`（hash 计算正确）
- 若修改 `_SIGNAL_RANKER_SYSTEM` 内容（哪怕一个字符），hash 值改变

## Notes

- SHA-256 前 12 位 hex 提供足够碰撞抵抗（16^12 ≈ 2.8×10^14 种可能）
- `PROMPT_CONTRACT_HASH` 被 `session_store.save_session()` 用于写入 `prompt_hash` 列
- 审计表可通过对比 hash 追溯 prompt 漂移时间点
