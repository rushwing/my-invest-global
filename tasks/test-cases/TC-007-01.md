## Objective

验证 `MacroStorage.upsert_indicators()` 能创建 `macro_indicators` 表，并在相同
PRIMARY KEY `(indicator_id, period_date, source)` 冲突时更新已有行，而不是插入重复行。

## Preconditions

- 使用 `MacroStorage(db_path=":memory:")`
- 测试不访问真实 `data/db/aidc.duckdb`

## Fixture

初始记录：

```python
{
    "indicator_id": "DGS10",
    "group_code": "K",
    "period_date": date(2026, 5, 15),
    "market_tz": "America/New_York",
    "value": 4.5,
    "value_unit": "percent",
    "value_raw": "4.5",
    "source": "fred",
    "realtime_start": None,
    "vintage_as_of": date(2026, 5, 16),
    "utc_ts": datetime(2026, 5, 16, 1, 0, tzinfo=UTC),
    "fetched_at": datetime(2026, 5, 16, 1, 0, tzinfo=UTC),
}
```

更新记录使用相同 `indicator_id`、`period_date`、`source`，但 `value=4.6`、
`value_raw="4.6"`、`fetched_at` 更晚。

## Steps

1. 构造 `storage = MacroStorage(":memory:")`
2. 调用 `storage.upsert_indicators([initial_record])`
3. 调用 `storage.upsert_indicators([updated_record])`
4. 查询 `macro_indicators` 中 `indicator_id="DGS10"` 的全部行

## Expected Result

- 两次 upsert 均返回 `1`
- 查询结果只有 1 行
- 该行 `value == 4.6`
- 该行 `value_raw == "4.6"`
- 该行 `fetched_at` 为更新记录中的较晚时间
