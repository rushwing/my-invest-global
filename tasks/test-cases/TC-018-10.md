## Objective

验证 `render_tab3()` 读取 `st.session_state["tab3_code"]` 后会清除该 key，避免后续 rerun 重复强制选择同一 ticker。

## Preconditions

- `app.components.tab3_stock.render_tab3` 可 import
- monkeypatch `st.session_state` 初始含 `{"tab3_code": "300308"}`
- mock 所有数据加载与 Streamlit 渲染调用

## Steps

1. 设置 `st.session_state["tab3_code"] = "300308"`。
2. 调用 `render_tab3()`。
3. 断言：
   ```python
   assert "tab3_code" not in st.session_state
   ```

## Expected Result

- `tab3_code` 被消费一次后清除
- 后续 rerun 不会重复使用旧预选值

- 与 TC-018-09 配套覆盖跳转预选生命周期
