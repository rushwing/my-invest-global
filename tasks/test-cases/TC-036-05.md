## Objective

验证 `70成本-低` / `70成本-高` / `集中度` 缺失时，`fetch_chip_summary` 按 REQ-036 fallback 规则生成 `ChipSummary`，不抛异常。

## Preconditions

- `fetch_chip_summary` 可 import
- mock `akshare.stock_cyq_em` 返回缺少 `70成本-低`、`70成本-高`、`集中度` 的 DataFrame
- DataFrame 仍包含关键字段：`获利比例`、`平均成本`、`90成本-低`、`90成本-高`

## Steps

1. 构造 mock DataFrame，删除可 fallback 的 3 个列。
2. patch `akshare.stock_cyq_em` 返回该 DataFrame。
3. 调用 `summary = fetch_chip_summary("688143")`。
4. 断言：
   ```python
   assert summary.range_70_lower == summary.range_90_lower
   assert summary.range_70_upper == summary.range_90_upper
   assert 0 <= summary.concentration <= 100
   ```

## Expected Result

- 正常返回 `ChipSummary`
- 70% 区间 fallback 到 90% 区间
- concentration 使用公式 fallback，且截断到 `[0, 100]`

## Notes

- 关键字段缺失仍应抛 `ChipDataUnavailable`，该行为可在 TC-036-04 或实现补充测试中覆盖
