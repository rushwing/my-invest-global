## Objective

验证 DuckDB 中没有指定 code 的 chip_summary 时，`load_and_analyze` 抛出 `ChipDataUnavailable(code=...)`。

## Preconditions

- `load_and_analyze` 可 import
- `ChipDataUnavailable` 从 `engine.agent.chip_fetcher` 可 import
- in-memory DuckDB 已初始化但没有 `code="000000"` 数据

## Steps

1. 创建空 in-memory DuckDB 并初始化 schema。
2. 调用：
   ```python
   with pytest.raises(ChipDataUnavailable) as exc:
       load_and_analyze("000000", 10.0, conn)
   ```
3. 断言：
   ```python
   assert "000000" in str(exc.value)
   ```

## Expected Result

- 抛出 `ChipDataUnavailable`
- 不返回空 `ChipAnalysis`

## Notes

- 此 TC 覆盖数据库缺数据路径
