## Objective

验证侧边栏保存持仓后调用 `st.cache_data.clear()` 与 `st.rerun()`，并且下一次 `load_latest_holdings()` 读取新保存的 `market_value`。

## Preconditions

- `app.sidebar.save_holdings` 可 import
- `app.data_loader.load_latest_holdings` 可 import
- 使用 `tmp_path` 或 monkeypatch 数据目录隔离 `data/agent_input/cn`
- monkeypatch `st.cache_data.clear` 与 `st.rerun`

## Steps

1. 构造 holdings DataFrame，其中 `current_price * quantity` 产生新的 `market_value`。
2. 调用保存按钮对应逻辑或 `save_holdings(editor_df, date)`。
3. 模拟 UI 保存后调用 `st.cache_data.clear(); st.rerun()`。
4. 下一次调用 `load_latest_holdings()`。
5. 断言新 DataFrame 的 `market_value` 等于保存值。

## Expected Result

- 保存文件写入最新 holdings CSV
- 缓存被清除并触发 rerun
- Tab 1 后续读取能看到新 `market_value`

## Notes

- 不要求真实 2 秒计时，可用同步单元测试证明刷新路径
