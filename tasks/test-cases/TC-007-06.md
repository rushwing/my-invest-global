## Objective

验证 `MacroStorage.upsert_regime()` 能写入并更新 `macro_regime` 表，
`get_regime_latest()` 返回最新 `as_of_date` 的完整状态字段。

## Preconditions

- 使用 `MacroStorage(db_path=":memory:")`

## Fixture

初始记录：

```python
{
    "as_of_date": date(2026, 5, 17),
    "capex_state": "green",
    "yield_curve_state": "normal",
    "risk_state": "risk_on",
    "composite_state": "green",
    "capex_as_of": "2026-Q1",
    "yield_as_of": date(2026, 5, 16),
    "computed_at": datetime(2026, 5, 17, 8, 0, tzinfo=UTC),
}
```

更新记录使用相同 `as_of_date`，但 `risk_state="neutral"`、
`composite_state="yellow"`、`computed_at` 更晚。

另插入一条更早 `as_of_date=date(2026, 5, 16)` 的记录，用于验证 latest 排序。

## Steps

1. 构造 `storage = MacroStorage(":memory:")`
2. 调用 `storage.upsert_regime(older_record)`
3. 调用 `storage.upsert_regime(initial_record)`
4. 调用 `storage.upsert_regime(updated_record)`
5. 调用 `latest = storage.get_regime_latest()`

## Expected Result

- `latest["as_of_date"] == date(2026, 5, 17)`
- `latest["capex_state"] == "green"`
- `latest["yield_curve_state"] == "normal"`
- `latest["risk_state"] == "neutral"`
- `latest["composite_state"] == "yellow"`
- `latest["capex_as_of"] == "2026-Q1"`
- `latest["yield_as_of"] == date(2026, 5, 16)`
- `latest["computed_at"]` 为更新记录中的较晚时间
