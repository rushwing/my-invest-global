## Objective

验证 `load_and_analyze("688143", 131.60, conn)` 能从 in-memory DuckDB 读取 chip_summary，并返回 `ChipAnalysis`。

## Preconditions

- `engine.agent.chip_analysis.load_and_analyze` 可 import
- `engine.data_agent.storage.init_db` 已创建 chip 表
- in-memory DuckDB 中预插入 `code="688143"` 的一行或多行 chip_summary

## Steps

1. 创建 in-memory DuckDB 并初始化 schema。
2. 插入 `688143` 的 chip_summary 行，至少包含 REQ-037 所需字段。
3. 调用 `analysis = load_and_analyze("688143", 131.60, conn)`。
4. 断言：
   ```python
   assert analysis.code == "688143"
   assert analysis.current_price == pytest.approx(131.60)
   assert analysis.date == "插入行日期"
   ```
5. 若插入多日数据，断言 `date=None` 时选择最新日期行。

## Expected Result

- 返回 `ChipAnalysis`
- 使用 DuckDB 中最新 chip_summary
- 派生信号与 `analyze_chip` 规则一致

## Notes

- 不需要读取 chip_distribution；本 TC 只覆盖 summary 读取路径
