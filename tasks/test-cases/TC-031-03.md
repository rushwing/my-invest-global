## Objective

验证 `action_code_label("hold_add")` 返回 `"持有加仓"`，且未知 code 如 `"unknown_code"` fallback 返回 `"unknown_code"`。

## Preconditions

- `app.pages.analysis.action_code_label` 可 import
- `app.pages.analysis.action_code_color` 可 import

## Steps

1. 调用 `action_code_label("hold_add")`
2. 调用 `action_code_label("stop_loss")`
3. 调用 `action_code_label("unknown_code")`
4. 调用 `action_code_color("strong_add")`
5. 调用 `action_code_color("unknown_code")`

## Expected Result

- `action_code_label("hold_add")` == `"持有加仓"`
- `action_code_label("stop_loss")` == `"止损"`
- `action_code_label("unknown_code")` == `"unknown_code"`（fallback to input）
- `action_code_color("strong_add")` == `"🟢"`
- `action_code_color("reduce")` == `"🔴"`
- `action_code_color("unknown_code")` == `"⚪"`

## Notes

- 6 个标准 action_code 全覆盖：`strong_add`, `hold_add`, `hold`, `reduce`, `stop_loss`, `take_profit_alert`
- Fallback 行为保证 UI 在 LLM 输出非标准 action_code 时不 crash
- `action_code_color` 的 fallback 符号为 `⚪`
