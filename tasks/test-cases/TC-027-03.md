## Objective

验证 `AnalysisState` 是一个 `TypedDict`，其 key 集合精确包含 `snapshot`, `kg_subgraph`, `rag_chunks`, `signals`, `reasoning`, `errors`, `session_id` 这 7 个字段。

## Preconditions

- `engine.agent.state.AnalysisState` 可正常 import

## Steps

1. 通过 `typing.get_type_hints()` 或 `.__annotations__` 获取 `AnalysisState` 的 key 集合
2. 构造一个最小合法 `AnalysisState` 实例（所有 7 个 key 赋初值）
3. 尝试用缺少 key 的 dict 构造（可选，验证类型检查器报错）

## Expected Result

- `set(AnalysisState.__annotations__.keys())` == `{"snapshot", "kg_subgraph", "rag_chunks", "signals", "reasoning", "errors", "session_id"}`
- `{"snapshot": mock_snap, "kg_subgraph": {}, "rag_chunks": {}, "signals": [], "reasoning": {}, "errors": [], "session_id": "test"}` 可以无异常赋值为 `AnalysisState` 变量
- 缺少任一 key 时，类型检查器（mypy/pyright）报错

## Notes

- `TypedDict` 在运行时不做强制校验，此 TC 主要验证类型注解的正确性
- LangGraph 的 `StateGraph(AnalysisState)` 依赖 TypedDict 的 key 定义做 graph schema
