## Objective

验证两个账户函数的行为：
- `load_holdings()` 对持有同一股票（603986）的两个账户进行聚合，合并行的 account 字段为两账户的合并值
- `load_holdings_raw()` 返回全部原始行，不做聚合

## Preconditions

- `data/agent_input/cn/holdings.yaml` 包含 8 条 holding，其中 603986 兆易创新分属华西证券和西南证券各一条
- `engine.portfolio.load_holdings` 和 `engine.portfolio.load_holdings_raw` 可 import

## Steps

1. 调用 `raw = load_holdings_raw()`：
   ```python
   assert len(raw) == 8
   codes_603986 = [h for h in raw if h.code == "603986"]
   assert len(codes_603986) == 2
   accounts_603986 = {h.account for h in codes_603986}
   assert accounts_603986 == {"华西证券", "西南证券"}
   ```

2. 按账户过滤 raw：
   ```python
   huaxi = [h for h in raw if h.account == "华西证券"]
   xinan = [h for h in raw if h.account == "西南证券"]
   assert len(huaxi) == 3   # 603986, 002384, 600150
   assert len(xinan) == 5   # 300308, 603986, 300394, 688820, 002957
   ```

3. 调用 `agg = load_holdings()`：
   ```python
   assert len(agg) == 7
   merged_603986 = next(h for h in agg if h.code == "603986")
   assert merged_603986.account == "华西证券+西南证券"
   ```

## Expected Result

- `load_holdings_raw()` 返回 8 行，各行 account 非空
- 过滤后华西 3 只，西南 5 只
- `load_holdings()` 返回 7 行，603986 的 account 为 `"华西证券+西南证券"`

## Notes

- 测试直接读取真实的 holdings.yaml（无 mock），验证文件格式和聚合逻辑一致性
- `_aggregate_holdings()` 的 account 合并逻辑：`"+".join(sorted(set(accounts)))`
