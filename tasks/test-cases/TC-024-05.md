## Objective

验证注入 `llm`、`schema`、`session_runner` 后，`KGQuerier.query()` 不触发真实
`_get_schema()` 或 `_run_cypher()`，从而可在单元测试中完全脱离 Neo4j。

## Preconditions

- 用 spy 或 patch 监控 `KGQuerier._get_schema`
- 用 spy 或 patch 监控 `KGQuerier._run_cypher`
- 注入 mock `llm`、schema 和 `session_runner`

## Steps

1. 调用 `KGQuerier(conn).query("问题", llm=mock_llm, schema="schema", session_runner=runner)`。
2. 检查 spy 调用情况。

## Expected Result

- `_get_schema()` 未被调用
- `_run_cypher()` 未被调用
- 只有注入的 `session_runner` 负责读取 mock 结果
