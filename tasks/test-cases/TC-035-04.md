## Objective

验证 `scripts.run_backtest` CLI 在不依赖真实网络的情况下，可通过 mock OHLCV 分别运行 `ma_cross` 和 `macd_cross`，并正常退出。

## Preconditions

- `scripts/run_backtest.py` 可通过 `python -m scripts.run_backtest` 执行
- CLI 支持 `--mock` flag，生成或注入不少于 26 行 OHLCV
- mock OHLCV 使用小写列名 `open/high/low/close/volume`，index 为 `DatetimeIndex`

## Steps

1. 使用 subprocess 执行 MA 策略：
   ```python
   result = subprocess.run(
       [
           sys.executable, "-m", "scripts.run_backtest",
           "--code", "300308",
           "--strategy", "ma_cross",
           "--start", "2024-01-01",
           "--mock",
       ],
       capture_output=True,
       text=True,
       check=False,
   )
   assert result.returncode == 0
   assert "total_return" in result.stdout
   assert "max_drawdown" in result.stdout
   ```
2. 使用 subprocess 执行 MACD 策略：
   ```python
   result = subprocess.run(
       [
           sys.executable, "-m", "scripts.run_backtest",
           "--code", "300308",
           "--strategy", "macd_cross",
           "--start", "2024-01-01",
           "--mock",
       ],
       capture_output=True,
       text=True,
       check=False,
   )
   assert result.returncode == 0
   assert "total_return" in result.stdout
   assert "max_drawdown" in result.stdout
   ```
3. 验证命令 stderr 不包含 traceback：
   ```python
   assert "Traceback" not in result.stderr
   ```

## Expected Result

- `ma_cross --mock` 命令退出码为 0
- `macd_cross --mock` 命令退出码为 0
- 两次运行均打印关键指标，例如 `total_return` 和 `max_drawdown`
- 两次运行都不访问 akshare，不依赖真实网络
- stderr 不包含 Python traceback

## Notes

- 此 TC 覆盖 CLI 集成与 mock 数据路径
- 真实 akshare 拉取路径可在手动 smoke test 中验证，不作为 CI 前置条件
