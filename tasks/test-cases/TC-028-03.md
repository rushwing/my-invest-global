## Objective

验证 `kg_retrieval(state, kg_querier=mock_querier)` 在 KG 查询正常返回时，将结果追加到 `kg_subgraph`。

## Preconditions

- `engine.agent.nodes.kg_retrieval` 可 import
- 构造 `AnalysisState`，其中 `snapshot.holdings` 含 1 个 `HoldingRow`（code="300308"）
- `state["kg_subgraph"]` 预填充 `{"300308": ["成本: 50.00"]}`（模拟 portfolio_analyzer 输出）
- mock `KGQuerier.query()` 返回 `KGQueryResult(error=None, answer="光通信")`

## Steps

1. 构造 `mock_querier`，其 `query("中际旭创（300308）属于哪些板块...")` 返回 `KGQueryResult(error=None, answer="光通信")`
2. 调用 `result = kg_retrieval(state, kg_querier=mock_querier)`

## Expected Result

- `result["kg_subgraph"]["300308"]` 包含 `"光通信"`
- `result["kg_subgraph"]["300308"]` 也保留原有的 `"成本: 50.00"`（追加而非覆盖）
- `result["errors"]` 为空或仅含 mock 错误之外的合法内容

## Notes

- `kg_retrieval` 不修改 state 本身——返回 partial dict，由 LangGraph merge
- 若 `kg_querier=None`，节点直接返回原 `kg_subgraph` 不做修改
