## Objective

验证当 `StockSignal.signals` 中不存在 `claimed_price` key 时，`verify_signals` 不产生任何 `ClaimViolation`。

## Preconditions

- `engine.agent.claim_verifier.verify_signals` 可 import

## Steps

1. 构造 `FrozenMarketSnapshot`，`price_snapshot={"300308": 100.0}`
2. 构造 `StockSignal(code="300308", ...)`，其中 `signals={}` 或不含 `claimed_price` key
3. 调用 `violations = verify_signals([sig], snapshot)`

## Expected Result

- `violations` == `[]`（空列表）
- 不抛任何异常

## Notes

- `claimed_price` 是可选字段——LLM 可能不输出此字段
- 验证逻辑只在 `if claimed is not None` 时才做偏差计算
- 这是宽松策略（不强制 LLM 输出 claimed_price），避免误报
