## Objective

验证手动宏观覆盖 UI 选择 `"red"` 后调用 `set_macro_state_override(MacroState.RED)`，随后执行 `st.cache_data.clear()` 和 `st.rerun()`。

## Preconditions

- `app.components.tab4_signals.render_macro_override_control` 或等价函数可 import
- monkeypatch `set_macro_state_override`
- monkeypatch `st.selectbox` 返回 `"red"`
- monkeypatch `st.button` 返回 `True`
- monkeypatch `st.cache_data.clear` 与 `st.rerun`

## Steps

1. 调用覆盖控件渲染函数。
2. 断言：
   ```python
   set_macro_state_override.assert_called_once_with(MacroState.RED)
   st.cache_data.clear.assert_called_once()
   st.rerun.assert_called_once()
   ```
3. 如使用真实临时文件路径测试 REQ-001 实现，则断言 JSON 仅需包含 `{"state": "red"}`。

## Expected Result

- 覆盖调用使用 `MacroState.RED`
- 缓存被清除
- 触发 Streamlit rerun

## Notes

- 当前 REQ-001 实现不写 `override` 字段，本 TC 不要求该字段
