## Objective

验证 `_refresh_crumb()` 在 TTL（24h）内第二次调用时直接返回缓存值，
不再发送任何 HTTP 请求。

## Steps

1. 构造 `YahooGlobalSource(rate_limiter)`
2. 用 `patch.object(src._session, "get")` mock session.get，返回有效 crumb
3. 第一次调用 `src._refresh_crumb()`
4. 第二次调用 `src._refresh_crumb()`

## Expected Result

- 两次返回值相同（同一 crumb 字符串）
- `session.get` 在第一次调用后不再被调用（第二次走缓存）
- `src._crumb_refreshed_at` 在第一次调用后被写入非 None 值
