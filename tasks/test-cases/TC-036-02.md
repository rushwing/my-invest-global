## Objective

验证 `store_chip_data(conn, summary)` 能将 `ChipSummary` 写入 in-memory DuckDB 的 `chip_summary` 和 `chip_distribution` 表。

## Preconditions

- `engine.data_agent.storage.init_db` 可初始化两张 chip 表
- `engine.agent.chip_fetcher.store_chip_data` 可 import
- 使用 `duckdb.connect(":memory:")`
- 准备包含两档以上 `bars` 的 `ChipSummary`

## Steps

1. 创建 in-memory DuckDB 连接并调用 `init_db(conn)`。
2. 构造 `ChipSummary(code="688143", date="2026-05-19", avg_cost=99.96, profitable_pct=0.9973, concentration=33.92, ...)`。
3. 调用 `store_chip_data(conn, summary)`。
4. 查询：
   ```sql
   SELECT * FROM chip_summary WHERE code='688143';
   SELECT * FROM chip_distribution WHERE code='688143';
   ```
5. 断言 summary 表返回 1 行，distribution 表返回行数等于 `len(summary.bars)`。
6. 验证核心字段类型和值：
   ```python
   assert row["avg_cost"] == pytest.approx(99.96)
   assert row["profitable_pct"] == pytest.approx(0.9973)
   ```

## Expected Result

- `chip_summary` 有对应 `(code, date)` 行
- `chip_distribution` 有对应多档筹码行
- 数值字段可从 DuckDB 正确查询

## Notes

- 使用 in-memory DuckDB，不读写项目真实数据库文件
