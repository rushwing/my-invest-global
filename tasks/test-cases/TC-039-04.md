## Objective

验证页面层 OCR radio 逻辑：当用户选择 `"截图解析（OCR）"` 但 `st.file_uploader` 返回 `None` 时，不调用 `parse_chip_screenshot`。

## Preconditions

- `app.pages.chip_panel.chip_panel_page` 可 import
- mock `st.radio` 返回 `"截图解析（OCR）"`
- mock `st.file_uploader` 返回 `None`
- mock `st.button` 返回 `False`，避免触发与本 TC 无关的分析路径
- mock 或 monkeypatch `parse_chip_screenshot`

## Steps

1. monkeypatch `app.pages.chip_panel.st` 为 mock 对象。
2. 设置：
   ```python
   st.radio.return_value = "截图解析（OCR）"
   st.file_uploader.return_value = None
   st.button.return_value = False
   ```
3. monkeypatch `parse_chip_screenshot` 为 mock。
4. 调用 `chip_panel_page()`。
5. 断言：
   ```python
   parse_chip_screenshot.assert_not_called()
   ```

## Expected Result

- 页面函数不抛异常
- 没有上传文件时不调用 OCR parser
- 不触发 `_render_chip_panel` / `load_and_analyze` / `fetch_chip_summary`
- REQ-038 未实现时也不影响本页面基础测试

## Notes

- 该 TC 测试页面层 OCR 控制流，不测试真实 OCR
