## Objective

验证 `kg_retrieval(state, kg_querier=mock_querier)` 在 KG 查询返回 error 时，错误信息写入 `errors`，`kg_subgraph` 不追加。

## Preconditions

- `engine.agent.nodes.kg_retrieval` 可 import
- 构造 `AnalysisState`，其中 `snapshot.holdings` 含 1 个 `HoldingRow`（code="300308"）
- `state["kg_subgraph"]` 预填充 `{"300308": ["成本: 50.00"]}`
- mock `KGQuerier.query()` 返回 `KGQueryResult(error="timeout")`

## Steps

1. 构造 `mock_querier`，其 `query(...)` 返回 `KGQueryResult(error="timeout")`
2. 调用 `result = kg_retrieval(state, kg_querier=mock_querier)`

## Expected Result

- `result["errors"]` 非空，包含 `"KG[300308]: timeout"`
- `result["kg_subgraph"]["300308"]` == `["成本: 50.00"]`（没有追加任何新内容）
- `len(result["kg_subgraph"]["300308"])` == 1（保持原有 1 条）

## Notes

- `kg_retrieval` 在 error 时不抛异常，保持 degraded mode 运行
- 此行为确保 KG 不可用时 pipeline 仍可继续到 `rag_retrieval` 和 `signal_ranker`
