## Objective

验证 `save_session(":memory:", state)` 写入后，`load_latest_session(":memory:")` 可以加载回 session_id 匹配、signals 长度相同的 `AnalysisState`。

## Preconditions

- `engine.agent.session_store.save_session`, `load_latest_session` 可 import
- 构造合法 `AnalysisState`（含 `FrozenMarketSnapshot` + 2 个 `StockSignal` + reasoning）

## Steps

1. 构造完整的 `AnalysisState`：
   ```python
   state: AnalysisState = {
       "snapshot": FrozenMarketSnapshot(
           session_id="test-roundtrip-001",
           holdings=(HoldingRow(...),),
           ...
       ),
       "kg_subgraph": {"300308": ["光通信"]},
       "rag_chunks": {"300308": ["年报摘要"]},
       "signals": [StockSignal(code="300308", ...), StockSignal(code="002230", ...)],
       "reasoning": {"300308": "AI 判断理由"},
       "errors": [],
       "session_id": "test-roundtrip-001",
   }
   ```
2. 调用 `save_session(":memory:", state)`
3. 调用 `loaded = load_latest_session(":memory:")`

## Expected Result

- `loaded` 不为 `None`
- `loaded["session_id"]` == `"test-roundtrip-001"`
- `len(loaded["signals"])` == 2
- `loaded["signals"][0]` 类型为 `StockSignal`
- `loaded["reasoning"]["300308"]` == `"AI 判断理由"`
- `loaded["errors"]` == `[]`
- `loaded["kg_subgraph"]["300308"]` 包含 `"光通信"`

## Notes

- DuckDB `:memory:` 隔离每次测试，不污染生产数据库
- `load_latest_session` 重建 placeholder `FrozenMarketSnapshot`（不存储完整快照）
- `INSERT OR REPLACE` 保证幂等（同 session_id 重跑覆盖）
