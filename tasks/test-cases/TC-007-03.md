## Objective

验证 `MacroStorage.upsert_capex()` 持久化 `capex_quarterly` 记录后，
`get_capex_quarters(company, n)` 返回最近 n 个季度，顺序为最新在前，且
`capex_usd` 单季值不被改写。

## Preconditions

- 使用 `MacroStorage(db_path=":memory:")`
- 测试数据使用 `company="MSFT"`，`source="sec_edgar"`

## Fixture

插入 4 条 CapEx 记录：

| fiscal_quarter | period_end | capex_usd |
| --- | --- | --- |
| 2025Q1 | 2025-03-31 | 3.0e9 |
| 2025Q2 | 2025-06-30 | 4.0e9 |
| 2025Q3 | 2025-09-30 | 3.5e9 |
| 2025Q4 | 2025-12-31 | 3.5e9 |

每条记录还包含 `cik="0000789019"`、`filing_form`、`source_hash`、`fetched_at`。

## Steps

1. 构造 `storage = MacroStorage(":memory:")`
2. 调用 `storage.upsert_capex(capex_records)`
3. 调用 `quarters = storage.get_capex_quarters("MSFT", 4)`

## Expected Result

- `upsert_capex()` 返回 `4`
- `len(quarters) == 4`
- `quarters[0]["fiscal_quarter"] == "2025Q4"`
- `quarters[0]["period_end"] == date(2025, 12, 31)`
- `quarters[0]["capex_usd"] == 3.5e9`
- 返回列表按 `period_end` 或等价的季度日期降序排列，最新季度在前
