## Objective

验证 `verify_signals()` 当 LLM 声称价格 999 但 `snapshot.price_snapshot` 记录为 100.0（偏差 899%，远超 `_PRICE_TOLERANCE=0.20`）时，返回 1 个 `ClaimViolation`。

## Preconditions

- `engine.agent.claim_verifier.verify_signals` 可 import
- `engine.agent.state.FrozenMarketSnapshot` 可 import
- `engine.schemas.StockSignal` 可 import（`extra="allow"` 支持 `signals` dict 传入 `claimed_price`）

## Steps

1. 构造 `FrozenMarketSnapshot`，`price_snapshot={"300308": 100.0}`
2. 构造 `StockSignal(code="300308", ...)`，其中 `signals={"claimed_price": "999"}`
   ```python
   sig = StockSignal(
       code="300308", name="中际旭创", category="白马股",
       technical_score=80, fundamental_score=85, sentiment_score=75,
       composite_score=80, action="持有", action_code="hold",
       reasoning="测试", signals={"claimed_price": "999"}
   )
   ```
3. 调用 `violations = verify_signals([sig], snapshot)`

## Expected Result

- `len(violations)` == 1
- `violations[0].code` == `"300308"`
- `violations[0].field` == `"price"`
- `violations[0].claimed_value` == `"999"`
- `violations[0].actual_value` == `"100.0"`

## Notes

- `_PRICE_TOLERANCE = 0.20` 表示允许 ±20% 偏差（正常日内波动）
- `price_snapshot` 中的价格来自 DuckDB 当日收盘价（非实时）
- `ClaimViolation` 不阻断 pipeline——由调用方写入 `state["errors"]`
