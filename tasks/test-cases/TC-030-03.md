## Objective

验证当 `analysis_sessions` 表为空时，`load_latest_session(":memory:")` 返回 `None` 而不抛异常。

## Preconditions

- `engine.agent.session_store.load_latest_session` 可 import
- DuckDB `:memory:` 全新连接（空表）

## Steps

1. 打开 `:memory:` DuckDB（不调用 `save_session`）
2. 直接调用 `result = load_latest_session(":memory:")`

## Expected Result

- `result` 为 `None`
- 不抛任何异常（无 `duckdb.Error`、`TypeError`、`IndexError` 等）
- `analysis_sessions` 表在查询过程中被自动创建（`CREATE TABLE IF NOT EXISTS`）

## Notes

- 空表场景常见于首次启动或数据库被清空后
- `load_latest_session` 使用 `fetchone()` 返回 `None` 时，函数直接返回 `None`
- 此 TC 验证 Dashboard 首次加载时不 crash
