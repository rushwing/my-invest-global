## Objective

验证 `load_ohlcv(ticker, days)` 在 DuckDB `price_history` 查询为空时调用 `fetch_akshare_history()`，随后调用 `write_ohlcv_to_duckdb()` 并返回按 date 升序的数据。

## Preconditions

- `app.data_loader.load_ohlcv` 可 import
- monkeypatch `duckdb.connect` 返回查询为空的 fake connection
- monkeypatch `fetch_akshare_history` 返回乱序 DataFrame
- monkeypatch `write_ohlcv_to_duckdb`

## Steps

1. 调用 `df = load_ohlcv("300308", days=180)`。
2. 断言：
   ```python
   fetch_akshare_history.assert_called_once_with("300308", 180)
   write_ohlcv_to_duckdb.assert_called_once()
   assert df["date"].is_monotonic_increasing
   ```

## Expected Result

- DuckDB miss 走 AkShare fallback
- fallback 数据写回 DuckDB
- 返回结果按日期升序

## Notes

- 不访问真实 AkShare 网络
