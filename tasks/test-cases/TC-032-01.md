## Objective

验证 `HoldingRow` 可以使用 `account` 字段构造，且 `account` 有正确的默认值（空字符串），使旧版 YAML 文件（无 account 字段）仍可正常加载。

## Preconditions

- `engine.schemas.HoldingRow` 可 import
- 准备合法的 HoldingRow 基础字段 dict（schema_version, date, code, name, cost_price, current_price, quantity, market_value, pnl_pct, pnl_amount, category, sector）

## Steps

1. 构造带 account 字段的 HoldingRow：
   ```python
   row = HoldingRow(**base_fields, account="华西证券")
   assert row.account == "华西证券"
   ```
2. 构造不带 account 字段的 HoldingRow（向后兼容）：
   ```python
   row = HoldingRow(**base_fields)
   assert row.account == ""
   ```
3. 验证 account 字段出现在 model_dump() 输出中：
   ```python
   assert "account" in row.model_dump()
   ```

## Expected Result

- 带 account 参数构造成功，值保留
- 不带 account 参数构造成功，默认值为 `""`
- `model_dump()` 包含 `account` 键

## Notes

- `HoldingRow` 使用 `extra="forbid"`，新字段必须在模型定义中声明
- 不需要 Streamlit / DuckDB / akshare
