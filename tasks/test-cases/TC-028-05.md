## Objective

验证 `run_analysis(snapshot, kg_querier=mock, retriever_fn=mock, llm=mock_llm)` 端到端运行 4 个节点，返回包含所有预期字段的 `AnalysisState`。

## Preconditions

- `engine.agent.runner.run_analysis` 可 import
- 构造合法的 `FrozenMarketSnapshot`（含 1 个 `HoldingRow`）
- mock `KGQuerier`：`query()` 返回 `KGQueryResult(error=None, answer="光通信龙头")`
- mock `retriever_fn`：接收 `(name, k)` 返回 `["年报摘要1", "年报摘要2"]`
- mock `ChatAnthropic`（或 monkeypatch `signal_ranker` 的 LLM 调用）：返回 JSON 含 1 个 `StockSignal`

## Steps

1. 构造所有 mock 注入
2. 调用 `result = run_analysis(snapshot, kg_querier=mock_kg, retriever_fn=mock_ret, llm=mock_llm)`

## Expected Result

- `result["session_id"]` == `snapshot.session_id`
- `result["signals"]` 长度 ≥ 1（mock LLM 返回的 signal 数量）
- `result["reasoning"]` 包含对应 code 的 key，值为非空字符串
- `result["kg_subgraph"]` 非空（含 portfolio_analyzer 和 kg_retrieval 的输出）
- `result["rag_chunks"]` 非空（含 rag_retrieval 的输出）
- `result["errors"]` 为 `list`（可能为空）
- `result` 包含全部 7 个 `AnalysisState` key

## Notes

- `run_analysis` 内部调用 `build_graph(...).invoke(initial_state)`
- 全 mock 注入保证无外部依赖（Neo4j、LanceDB、Anthropic API）即可运行
- mock LLM 需返回符合 `_SIGNAL_RANKER_SYSTEM` 格式的 JSON 数组
