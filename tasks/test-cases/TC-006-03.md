## Objective

验证 `SECEdgarSource.fetch_capex_quarterly()` 构造的 URL 包含 `CIK{cik}`
格式（如 `CIK0000789019`），而非裸数字。

## Preconditions

- HTTP 请求已 mock（捕获实际请求的 URL）

## Steps

1. 用 `unittest.mock.patch.object(SECEdgarSource, "_get")` 捕获调用参数，
   返回最小 companyfacts fixture（含 1 条 Q1 记录）
2. 调用 `fetch_capex_quarterly("0000789019")`
3. 取 mock 调用的第一个位置参数（URL）

## Expected Result

- `captured_url` 包含 `"CIK0000789019"` 子串
- `captured_url` 不包含 `"/0000789019.json"` 或 `"/0000789019"` 不带 CIK 前缀的路径
