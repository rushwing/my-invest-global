## Objective

验证 `fetch_technicals(codes)` 在 mock akshare DataFrame 下，能正确计算并返回 MA5/MA10/MA20/MA30/MACD_DIF/MACD_DEA/MACD_BAR/RSI14 各键的 float 值。

## Preconditions

- `engine.agent.technical_fetcher.fetch_technicals` 可 import
- `unittest.mock.patch` 可用于 mock `akshare.stock_zh_a_hist`
- 构造一个含至少 30 行 OHLCV 的 DataFrame（列名：收盘/开盘/最高/最低/成交量），确保 MA/MACD/RSI 有足够数据计算

## Steps

1. 构造 mock DataFrame（30 行，Close 从 100 线性涨到 130）：
   ```python
   import numpy as np
   import pandas as pd
   n = 30
   mock_df = pd.DataFrame({
       "收盘": np.linspace(100, 130, n),
       "开盘": np.linspace(99, 129, n),
       "最高": np.linspace(101, 131, n),
       "最低": np.linspace(98, 128, n),
       "成交量": [1_000_000] * n,
   })
   ```

2. patch `akshare.stock_zh_a_hist` 返回 mock_df，调用：
   ```python
   with patch("akshare.stock_zh_a_hist", return_value=mock_df):
       result = fetch_technicals(["300308"])
   ```

3. 验证返回值：
   ```python
   assert "300308" in result
   tech = result["300308"]
   for key in ["MA5", "MA10", "MA20", "MA30", "MACD_DIF", "MACD_DEA", "MACD_BAR", "RSI14"]:
       assert key in tech
       assert isinstance(tech[key], float)
   ```

## Expected Result

- `result["300308"]` 包含所有 8 个键
- 所有值为 float（不是 NaN，因为数据足够）
- MA5 > MA10 > 0（上升趋势 mock 数据）

## Notes

- pandas-ta 列名规律：`SMA_5`, `MACD_12_26_9`, `MACDs_12_26_9`, `MACDh_12_26_9`, `RSI_14`
- akshare 列名为中文，`fetch_technicals` 在计算前做 rename
