## Objective

验证 mock LLM 生成含 `MERGE` 的写 Cypher 时，`KGQuerier.query()` 在任何 DB 调用前阻断。

## Preconditions

- 注入 mock `llm`，生成 `"MERGE (s:Stock {code: '300308'}) RETURN s"`
- 注入 `session_runner` mock

## Steps

1. 调用 `KGQuerier(conn).query("创建股票", llm=mock_llm, schema="schema", session_runner=runner)`。
2. 检查返回结果和 `session_runner` 调用次数。

## Expected Result

- `result.error == "read_only_violation"`
- `result.cypher_used` 包含 `"MERGE"`
- `result.answer == ""`
- `result.raw_results == []`
- `session_runner` 从未被调用
