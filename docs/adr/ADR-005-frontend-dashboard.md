# ADR-005：前端仪表盘选型

- **状态**：已采纳
- **日期**：2026-05-13
- **决策者**：Daniel Wong

---

## 背景

分析引擎产出 JSON 信号和 Markdown 策略简报后，需要一个可视化界面：
- 展示持仓总览和仓位偏差
- 每日策略简报（LLM 输出）
- 个股技术图（MACD/RSI/布林带）
- 信号仪表盘（宏观状态 + 紧缺度矩阵）

约束：
- 个人使用为主，无需考虑多用户并发
- 与 Python 分析引擎紧密集成（同一进程或本地调用）
- 开发效率优先，快速迭代
- 参考 FinceptTerminal 的多面板布局理念

---

### 各方案详细分析

**Streamlit：**
- Pros：5 分钟出原型；`st.dataframe` 直接展示 pandas DataFrame；Plotly 图表一行代码；`st_autorefresh` 支持文件变化触发刷新；社区生态最活跃；最新版本（1.32+）支持多页面和自定义组件
- Cons：深度定制 UI 能力弱（不如 React）；全局状态管理比较奇特；多用户场景性能差

**Gradio：**
- Pros：极速原型；AI demo 生态好
- Cons：为 ML 模型演示设计，金融表格和 MACD 图表支持弱；布局能力差

**Textual TUI：**
- Pros：终端运行，无需浏览器；Python 原生；Rich 生态好
- Cons：图表能力有限（无法显示 MACD 等连续曲线图）；学习曲线较高

**FinceptTerminal：**
- Pros：专业金融终端，布局最完善
- Cons：C++/Qt6 实现，Python 只是嵌入脚本；维护成本极高；当前 /Users/danielwong/Dev/FinceptTerminal 几乎只有 .git，源码不完整；不建议 fork 作为基础
- **结论：参考其 UX 设计和多面板布局理念，不作为技术选型**

**Dash（Plotly）：**
- Pros：金融图表能力最强（蜡烛图、技术指标图）；布局灵活
- Cons：比 Streamlit 开发速度慢 3-5 倍；Callback 系统学习曲线陡

---

## 决策

**选定：Streamlit（v1.32+）**

参考 FinceptTerminal 的多面板布局理念，使用 Streamlit 的 Tab + Column 布局实现。

## 扩展路径

| 当前 | 升级条件 | 升级方案 |
|---|---|---|
| Streamlit | 需要更复杂交互或多用户 | Dash（保留 Plotly 图表，改用 Callback） |
| Streamlit | 团队协作，需要权限管理 | 迁移到 React 自研（API 层不变） |
| 本地运行 | 需要随时随地访问 | 部署到 Mac mini/NAS 上跑 Streamlit；或 Railway/Render |
| Plotly 技术图 | 需要更专业的金融图 | 集成 lightweight-charts（TradingView 同款） |

## 后果

- ✅ 开发极快：Streamlit 几天内出完整仪表盘原型
- ✅ Python 原生：分析引擎与前端同进程，数据传递零开销
- ✅ Plotly 图表满足所有技术分析可视化需求
- ⚠️ 深度 UI 定制受限：Streamlit 的布局系统不如 React 灵活
- ⚠️ 多用户并发时性能下降（个人使用无问题）
- ⚠️ FinceptTerminal 的部分交互模式（如实时 tick 行情）Streamlit 无法复现——本工具不需要实时行情，可接受
