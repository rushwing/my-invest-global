# ADR-004：分析引擎设计（技术 + 基本面 + 情绪三信号融合）

- **状态**：已采纳
- **日期**：2026-05-13
- **决策者**：Daniel Wong

---

## 背景

需要设计一个分析引擎，输入持仓数据 + Codex Agent 输出，产生可解释的操作建议。

核心要求：
- **可解释**：每条建议都能溯源到具体信号（不能是黑盒）
- **AI 基建专属**：考虑该板块的特殊信号（隔夜美股先行效应、CapEx 周期、BIS 管制）
- **Token 经济**：不在分析引擎里做实时 web search，消费 Codex Agent 预处理的结构化数据
- **人工最终决策**：输出是建议，不是指令

---

## 可选方案

| 方案 | Pros | Cons |
|---|---|---|
| **规则引擎 + LLM 合成（选定）** | 可解释；每步可审查；LLM 只做语言合成；调试方便 | 规则权重需要人工校准；两个组件维护 |
| 纯规则引擎（无 LLM） | 最简单；完全可解释；零 API 成本 | 输出是数字/表格，不够人性化；缺乏上下文融合 |
| 纯 LLM 推理（直接 prompt） | 最灵活；能理解上下文 | Token 开销巨大；不稳定（幻觉风险）；不可解释 |
| LightGBM 因子模型（Qlib 式） | 数据驱动；自动学习权重 | 需要历史标注数据（50 只股票池极易过拟合）；黑盒 |
| 贝叶斯更新模型 | 理论优雅；概率输出 | 实现复杂；先验分布设定困难 |

---

## 决策

**三信号规则引擎打分 → Claude API 自然语言合成**

---

## 三信号设计

### Signal A：技术分析（权重 30%）

**数据来源：** 历史价格数据（Tushare/AkShare 拉取，缓存在 Parquet）

**指标计算（pandas-ta）：**

| 指标 | 参数 | 信号解读 | 分值贡献 |
|---|---|---|---|
| MACD | 快12/慢26/信号9 | 日线金叉 +15分，死叉 -15分；周线优先级更高（×1.5） | 0-30 |
| RSI | 14日 | <30 超卖（+10）；>70 超买（-10）；30-70 中性（0） | 0-10 |
| 布林带 | 20日，2σ | 价格在下轨附近（+10）；上轨附近（-5）；中轨（0） | 0-10 |
| 量价 | 量比 5日均量 | 放量上涨（+10）；缩量下跌（-5）；其他（0） | 0-10 |
| 周线趋势 | 13周均线 | 价格上穿 13 周均线（+10）；下穿（-10） | 0-10 |

**技术评分 = Σ(上述各项) → 归一化到 0-100**

**A 股特殊处理：**
- 停牌日：技术信号标记为"暂停"，不计入当日综合分
- 涨停日：RSI/布林带信号暂时失效（追涨不适用，需 T+1 再评估）

---

### Signal B：基本面（权重 40%）

**数据来源：** 研报静态数据（data_china_aidc.md，季度更新）+ 持仓成本

| 因子 | 计算方式 | 满分 | 说明 |
|---|---|---|---|
| 产品紧缺度 | Tier-1=25, Tier-2=18, Tier-3=10, Tier-4=3 | 25 | 来自 data_china_aidc.md |
| 目标价上行空间 | (目标价-现价)/现价 × 100，上限 30 | 30 | 机构目标价，研报数据 |
| Q1 财报超预期 | 营收增速×0.3 + 利润增速×0.7，归一化 | 25 | 来自财报摘要 |
| 行业排行 | #1=20, #2=13, #3=7, 其他=2 | 20 | 细分板块内排名 |

**基本面评分 = Σ(上述各项) → 归一化到 0-100**

**特殊规则：**
- 若当前价格已超过目标价 → 目标价因子得分清零，触发"目标价警报"
- 若持仓亏损 >20%（弹性股）→ 触发强制止损提示，覆盖其他信号

---

### Signal C：情绪/新闻（权重 30%）

**数据来源：** Codex Agent 输出的 daily_news JSON + kol_digest JSON

| 因子 | 来源 | 满分 | 说明 |
|---|---|---|---|
| 隔夜美股先行指标 | daily_news（is_overnight_us=true） | 40 | NVDA/AVGO/VRT/ANET/MU 涨跌幅加权 |
| 相关新闻情绪 | daily_news（按 tickers_mentioned 过滤） | 30 | positive=满分；negative=0；neutral=15 |
| 大V 情绪 | kol_digest（情绪+相关标的） | 20 | 同上逻辑 |
| 机构研报更新 | weekly_report | 10 | 有新研报且正面=10；无=5；负面=0 |

**隔夜美股先行指标计算：**
```python
us_weights = {
    "NVDA": 0.35,   # 对光模块/PCB 链影响最大
    "AVGO": 0.20,   # 网络 ASIC，对交换机链有影响
    "VRT":  0.20,   # 液冷，对散热链有影响
    "ANET": 0.15,   # AI 以太网，对网络设备有影响
    "MU":   0.10,   # HBM 内存，对存储/澜起科技有影响
}
overnight_score = sum(
    us_weights[t] * clip(pct_change[t] * 10, -40, 40) + 20
    for t in us_weights
)
# 归一化到 0-40
```

### 融合评分

```
综合分 = A × 0.30 + B × 0.40 + C × 0.30

操作建议映射：
  ≥ 75  → strong_add    强烈建议加仓
  60-74 → hold_add      建议持有/小幅加仓
  40-59 → hold          观望
  < 40  → reduce        考虑减仓
  
特殊覆盖（优先级高于综合分）：
  弹性股持仓亏损 > 20%     → stop_loss     止损提示
  标的距目标价上行空间 < 5% → take_profit_alert  目标价警报
  宏观 CapEx 状态=Red      → 全局仓位警告，弹性股一律降为 hold
```

### LLM 合成层

**输入 Prompt 结构（精简版）：**

```
你是 AI 基建股策略顾问，以下是今日分析数据：

【持仓概况】
总市值：XXX万，白马股 68%，弹性股 32%，目标 67%/33%

【各标的评分】
中际旭创 300308：综合分 83（技术70/基本面92/情绪80）→ 持有/小幅加仓
澜起科技 688008：综合分 78 ...
...

【今日关键信息】
- 隔夜：NVDA +8.2%（Q1 财报超预期，Blackwell 出货加速）
- 相关新闻：[摘要1], [摘要2]
- 大V 观点：[韭圈儿摘要]

【请生成：】
1. 今日市场判断（3句话，重点在 AI 基建板块情绪）
2. 持仓操作建议（逐标的，每条 ≤2 句）
3. 需要注意的风险（1-2条）

输出格式：Markdown，简洁专业，中文。
```

**token 预算控制：**
- 输入 prompt ≤ 3000 tokens（数据结构化，避免冗长）
- 输出 ≤ 800 tokens（策略简报，非长文）
- 每日运行一次（非实时，控制 API 成本）

## 扩展路径

| 扩展 | 触发条件 | 实现方式 |
|---|---|---|
| 个股专属权重 | 积累 3-6 个月观察数据 | 为每只股票微调 A/B/C 权重 |
| LightGBM 因子模型 | 积累 2 年+标注数据 | 替换规则引擎，保留 LLM 合成层 |
| 本地 LLM（Ollama） | API 成本过高或需离线 | 替换 Claude API，接口已抽象 |
| 事件研究模块 | 需要量化财报/政策 impact | 独立 event_study.py 模块 |

---

## 后果

- ✅ 每条建议完全可溯源（技术分 X = MACD 金叉 +15 + RSI 超卖 +10...）
- ✅ 隔夜美股先行效应量化：NVDA +8% → 光模块链情绪分上调 ×5
- ✅ Token 经济：LLM 只做最后合成，不做数据采集，每日消耗 <4000 tokens
- ⚠️ 规则权重（30%/40%/30%）需要运行 3-6 个月后根据准确率校准
- ⚠️ 情绪信号在市场恐慌时可能失效（极端情绪不适用规则引擎）
