# 系统架构文档

> AI 基建股持仓顾问工具 — 完整架构说明  
> 版本：v0.1 | 日期：2026-05-13

---

## 设计原则

1. **数据采集与分析解耦**：Codex Agent 负责所有数据采集，本工具只消费结构化文件，避免 token 冗余开销
2. **推荐而非执行**：输出操作建议，人工做最终决策，杜绝自动交易风险
3. **可解释优于黑盒**：规则引擎打分（可审查每一步推导），LLM 只做自然语言合成
4. **本地优先，免服务器**：DuckDB + Parquet 在本地运行，无云基础设施依赖
5. **扩展性预埋**：文件接口契约版本化，未来可零改动升级为 Redis 队列

## 扩展路径

| 当前 | 升级路径 | 触发条件 |
|---|---|---|
| 文件系统接口 | Redis Streams | Agent 数量增加，需实时推送 |
| 本地 DuckDB | ClickHouse | 数据量 >1 亿行，多用户并发 |
| Streamlit | React 自研 | 团队规模扩大，需深度定制 |
| 规则引擎打分 | LightGBM（需历史标注） | 积累 >2 年标注数据后 |
| Claude API 合成 | 本地 Ollama（Qwen3-32B） | 需要离线运行或降低 API 成本 |
