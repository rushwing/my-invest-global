# 研报生成 Skill

## 路径判断规则

读取用户的完整 prompt，**优先匹配 Path-B 触发词**，未命中则走 Path-A。

| 路径 | 触发条件 | 特征 |
|---|---|---|
| **Path-B 本地 AIDC** | 含 `AIDC` 或 `aidc` 或 `算力研报`；**或** 含（`本地`/`刷新`/`更新`/`从本地`）**且同时**含（`AIDC`/`aidc`/`算力`/`数据中心`） | 固定领域、固定数据源、固定结构 |
| **Path-A 网络搜索** | 不满足 Path-B 条件的所有其他情况 | 任意主题，依赖 WebSearch |

> **路由示例**：`从本地数据生成新能源产业链研报` → `本地` 命中但无 AIDC 信号 → **Path-A**；`刷新AIDC研报` → `刷新` + `AIDC` → **Path-B**；`生成算力研报` → `算力研报` 直接命中 → **Path-B**。

---

## Path-A：从零开始生成研报（联网搜索）

### 适用场景
用户想研究一个本地数据库中没有覆盖的新主题：新行业、新公司、临时调研需求。

### 执行步骤

**Step 1 — 明确主题**
如果 prompt 中未指定具体标的或行业，先向用户确认：
> "请确认研报主题（行业/板块/个股），以及侧重方向（中国市场/美国市场/中美对比）？"

**Step 2 — 宏观搜索（并行执行）**
使用 WebSearch 并行搜索以下维度，每项 1-2 条高质量来源：
- `[主题] 产业链 2025 2026 最新研报 行业综述`
- `[主题] 龙头公司 业绩 资本支出 景气度`
- `[主题] 技术趋势 供需格局 稀缺节点`
- `[主题] 投资风险 估值 PE 分位数`

**Step 3 — 结构化输出**
按以下固定结构生成 Markdown 研报：

```
# [主题] 投资研究报告
> 生成日期：{today} | 数据来源：联网搜索 | 仅供研究参考

## 一、宏观与产业判断
（当前所处周期位置、主要驱动力、不可逆逻辑）

## 二、产业链稀缺节点矩阵
（仿照 methodology.md 四级稀缺矩阵格式，Tier-1 ~ Tier-4）

## 三、核心标的评级
（表格：标的 | 市场 | 稀缺等级 | 当前估值 | 目标价 | 评级 | 核心逻辑）

## 四、建仓策略建议
（参照时寒冰金字塔：核心仓 40% / 卫星仓 35% / 机动仓 25%）

## 五、风险提示
（政策风险 / 技术替代风险 / 估值风险 / 汇率风险）

## 六、关键监控指标
（触发加减仓的具体信号，参照 methodology.md 信号灯框架）
```

**Step 4 — 可选输出**
询问用户是否需要输出 HTML 文件（如需要，参照 Path-B 的 HTML 渲染方式）。

## Path-B：从本地数据生成 AIDC 研报

### 适用场景
基于项目内的固定本地数据，快速刷新 AIDC 产业链研报。领域、结构、数据源全部固定。

### 固定数据源（按顺序全部读取）

| 文件 | 内容 | 读取方式 |
|---|---|---|
| `methodology.md` | 时寒冰方法论框架、稀缺矩阵、建仓策略、信号灯 | 全文读取，作为分析框架 |
| `data_china_aidc.md` | A 股 AIDC 产业链核心标的（数量以文件头部注释为准）：股价/PE/涨跌幅来自腾讯行情实时刷新，单平铺表格，A 股目标价列全部留空 | 全文读取 |
| `data_us_aidc.md` | 美股 AIDC 核心标的：股价、PE、Q1 财报、券商目标价、评级 | 全文读取 |
| `data/processed/signals_*.json` | 最新三信号评分（如存在） | 读取最新日期文件 |
| `data/agent_input/daily_news_*.json` | 最新 Codex 新闻摘要（如存在） | 读取最新日期文件 |

若 `data/processed/` 或 `data/agent_input/` 下无文件，跳过对应步骤，不报错。

**目标价数据规范（重要）**
- A 股目标价：`data_china_aidc.md` 中该列全部留空，表示"尚无已核验目标价"，**不等于目标价已达到**。`scripts/refresh_aidc_data.py` 的 `VERIFIED_TARGETS` 字典是唯一可信来源——只收录研报日期 ≥ 2026-01-01 且已人工核验的券商目标价，当前为空。
- 读到空目标价 → 上行空间**未知**，该因子在评分中置为中性（不加分也不减分），**不触发 `take_profit_alert`**，不得估算或引用旧数据补填。
- `take_profit_alert` 仅在 `VERIFIED_TARGETS` 中存在该标的的目标价、且当前价格 ≥ 目标价 × 0.95 时才可触发。
- 美股目标价：`data_us_aidc.md` 中来源已标注券商和日期，可直接使用。

### 执行步骤

**Step 0 — 可选：刷新行情数据**
询问用户是否需要先执行 `uv run refresh-aidc-data` 以拉取最新腾讯行情（价格/PE/涨跌幅）再生成研报。如用户确认，运行该命令后再执行后续步骤。

**Step 1 — 读取所有数据源**
并行读取上表中所有文件，记录每个文件的数据截止日期，取最晚日期作为报告日期。

**Step 2 — 宏观判断（依据 methodology.md 信号灯）**
从数据中提取四大云厂商资本支出状态，判断当前信号灯颜色（绿/黄/红），对应弹性股目标仓位：
- 绿灯（资本支出 QoQ ≥ +5%）：弹性股目标 38-40%
- 黄灯（QoQ -5% ~ +5%）：弹性股目标 33%
- 红灯（QoQ < -10%）：弹性股目标 20%

**Step 3 — 产业链稀缺矩阵更新**
按 methodology.md 的四级稀缺体系重新整理，结合 data_china_aidc.md 和 data_us_aidc.md 的最新评分和财报摘要，标注变化：
- 哪些标的稀缺度升级/降级
- 哪些 Q1 财报超预期/不及预期
- 哪些估值出现信号（PE > 历史峰值 1.5 倍 → 黄色预警）

**Step 4 — 信号融合评级**
如存在 `signals_*.json`，将三信号评分（技术 A × 0.30 + 基本面 B × 0.40 + 情绪 C × 0.30）映射为操作建议：

| 综合分 | 操作建议 |
|---|---|
| ≥ 75 | 强烈建议加仓 |
| 60–74 | 建议持有/小幅加仓 |
| 40–59 | 观望 |
| < 40 | 考虑减仓 |

如不存在 signals 文件，基于基本面评级和稀缺度给出定性建议。

**Step 5 — 生成 Markdown 研报**
固定结构：

```
# AIDC 产业链投资研究报告
> 数据截止：{最新文件日期} | 方法论：时寒冰全球视野框架 | 仅供投资研究参考

## 一、宏观信号灯：当前状态
（云厂商资本支出信号 | 当前灯色 | 弹性股目标仓位）

## 二、产业链稀缺节点全景
（四级稀缺矩阵表格，突出本期变化）

## 三、中国市场：A 股核心标的综评
（按当前轮动热度排序：光通信 → 内存/半导体 → 算力/芯片 → PCB → 上游材料
 → AI服务器 → 设备封测 → 散热/液冷 → 电源系统；AI网络板块已标记 Deprecated 不纳入推荐。
 每版块列出：龙头标的当前状态 / Q1 财报关键数据 / 操作建议 + 信号来源）

## 四、美国市场：美股核心标的综评
（按半导体/服务器/网络/液冷电源/光互联 分版块）

## 五、中美产业链对比与联动
（中国国产替代进度 | 美股领先指标对 A 股的传导）

## 六、建仓策略建议
（时寒冰金字塔：当前宏观状态下的核心仓/卫星仓/机动仓推荐标的）

## 七、风险提示
（当前主要风险：估值透支 / 资本支出缩减 / 技术替代 / 汇率）

## 八、关键监控指标仪表盘
（下次触发加减仓的具体信号，含阈值）
```

**Step 6 — 输出文件**
- Markdown：直接输出到对话
- HTML（可选）：写入 `{topic}_report.html`（仿照现有 `aidc_report.html` 样式：深色主题、响应式表格、颜色标注评级、含产业链 SVG 流向图）

询问用户：`是否同时生成 HTML 文件保存到项目根目录？`

---

## 通用输出规范

- 所有表格保持 `data_china_aidc.md` / `data_us_aidc.md` 的列结构一致性
- 评级用统一标记：`🟢 强烈推荐` / `🔵 推荐` / `🟡 中性` / `🔴 减仓观察`
- 信号来源必须标注（数据文件名 + 字段）
- 数据截止日期在报告标题下方显著标注
- 不生成投资建议免责声明之外的任何法律文本

---

## 扩展预留

- **新增市场**：在 Step 1 数据源表中增加对应 `data_{market}_{sector}.md` 文件即可，无需修改 skill 结构
- **新增信号**：在 `methodology.md` 中扩展信号灯维度，Path-B Step 2 自动继承
- **新增输出格式**：在 Step 6 增加格式选项（如 Word、Excel），不影响已有路径
- **MCP 数据源接入**：未来若 `tools/mcp/market-data/` 提供实时行情 MCP，Path-B Step 1 可替换静态文件读取为 MCP 工具调用，路径判断和报告结构不变
